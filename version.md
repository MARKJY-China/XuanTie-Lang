玄铁版本（XuanTie SemVer）框架：
版本 SemVer：主版本.次版本.修订号.测试号

主版本（Major）：破坏性变更/产品里程碑
次版本（Minor）：向后兼容功能/部分新功能/大更新
修订号：向后兼容缺陷修复/Bug 修复/小更新

## v0.15.0 (2026-04-18)
- **里程碑：实现复杂数据结构与面向对象基础 (OOP & Data Structures)**：
    - **原生集合支持**：
        - **数组 (Array)**：支持 `[1, 2, 3]` 字面量定义、下标索引读取 `列表[0]` 及下标赋值 `列表[0] = 值`。底层对接运行时 `xt_array` 接口。
        - **字典 (Dict)**：支持 `{"键": 值}` 字面量定义。底层对接运行时 `xt_dict` 哈希表实现。
    - **面向对象系统 (OOP)**：
        - **型定义 (Class Definition)**：支持 `型 名称 { 设 属性 ... 函 方法 ... }` 语法，实现了属性存储与方法封装。
        - **对象实例化 (Instantiation)**：引入 `造` 关键字，支持 `造 类型(参数)` 语法。实例化时会自动初始化属性，并调用具名的 `造` 方法作为构造函数。
        - **动态方法分发 (Dynamic Dispatch)**：实现了方法指针注入机制。在实例化阶段将类方法指针存入对象字典，调用时通过 `对象.方法名()` 进行动态查找与执行。
        - **`此` (this) 指针支持**：自动为类方法注入 `此` 参数，支持在方法内部通过 `此.成员` 访问实例状态。
    - **编译器前端与稳健性增强**：
        - **算符优先解析器重构**：完善了 Pratt Parsing 算法，新增对点操作符 `.` 和方括号 `[]` 的嵌套解析支持。
        - **死循环防御机制**：在 `解析型定义`、`解析字典` 等循环解析逻辑中引入了强制 Token 消费与容错处理，彻底解决了因非法语法导致的编译器内存溢出与卡死问题。
    - **ARC 内存管理加固**：
        - 针对数组/字典设值、方法调用及临时对象生成，精细化了 `xt_retain` 与 `xt_release` 的插入逻辑。
        - 确保临时生成的对象（如字面量、表达式中间值）在存入容器或完成传递后被及时释放，显著提升了自举编译器的内存安全性。

## v0.14.5 (2026-04-17)
- **编译后端深度优化：类型推导与原生去标签化 (Type Inference & Unboxing)**：
    - **原生整数类型 (`raw_i64`)**：在 Go 版种子编译器 (`compiler/llvm.go`) 和自举编译器 (`编译.xt`) 的代码生成器中引入了代表底层未装箱整数的 `raw_i64` 内部类型。
    - **零开销算术运算**：针对字面量整数、算术运算 (`+`, `-`, `*`, `/`) 及比较运算 (`<`, `>`, `==`) 等，后端不再将其转换为带标签的指针（Tagged Pointers）。相反，编译器会通过新增加的 `ensureRawI64` 直接剥离标签，使用 O(1) 级别的原生 LLVM `add`, `sub`, `icmp` 等指令完成纯 CPU 寄存器层面的数学运算。
    - **循环计数器优化**：对 `for` 循环的底层指令进行了重构，提取数组/字符串长度及索引计数时全面切至原生整数运算，显著提升了紧密循环的执行性能。
    - **智能自动装箱 (`ensureI64`)**：只有当整数需要作为参数传递给类方法、动态对象或用于打印时，系统才会自动进行装箱 (`shl 1` 并 `or 1`)，真正实现了“需要时才装箱”的性能跃迁。

## v0.14.4 (2026-04-17)
- **自举架构深度修复与完善**：
    - **方法调用返回类型修复**：修正了 `llvm.go` 中编译方法调用 (`compileMethodCallExpression`) 丢失返回类型的问题，现在方法调用会正确保留实例类型 (`objClass`)，保障链式调用不走兜底搜索。
    - **参数所有权降级**：统一了 `llvm.go` 中的方法参数传递逻辑。现在将所有传递的实参统一确认为 `i64` 寄存器（`XTValue`）并释放，消除了原先双重转换（`convertToObj`）带来的生命周期管理歧义。
    - **转换指令纠正**：修复了自举源码 `编译.xt` 中 `确保I64` 方法里的指针转换错误。将原先错误的 `inttoptr` 修正为合法的 LLVM 指令 `ptrtoint`，确保指针对象能够正确转换为 64 位整数参与 ARC 计算。
    - **作用域堆栈泄漏修复**：修复了 `编译.xt` 中 `出作用域()` 方法未弹出栈顶元素的严重逻辑漏洞。现在单层退出时，会通过重构数组正确移除已释放的作用域层，彻底解决了深层嵌套导致的内存重复释放问题。
    - **词法消费时机修正**：移除了 `入口.xt` 中错误提前调用的 `词法.执行分析()`，使语法分析器能够按需驱动词法流，修复了编译器输出为空的异常状态。
    - **技术白皮书发布**：新增并发布了《玄铁自举技术架构白皮书.md》，全面总结并固化了玄铁自举阶段的架构设计、ARC 内存管理以及类型系统的底层实现规范。

## v0.14.3 (2026-04-17)
- **运行时性能优化：对象内存池 (Memory Pooling)**：
    - **定长对象池 (Free-list Pool)**：在 C 运行时 `xt_runtime.c` 中引入了针对小尺寸结构体（如 `XTString`, `XTArray`, `XTInstance`, `XTResult`, `XTDict` 基础结构）的空闲链表内存池。池最大支持 128 字节，分为 8 个槽位。
    - **分配与回收策略**：`xt_malloc` 和 `xt_release` 现在会优先尝试从对应大小的内存池中获取和归还对象内存。只有在池空或对象过大时才会 fallback 到系统底层的 `malloc`/`free`。
    - **重用安全 (memset)**：从池中取出的重用对象会通过 `memset` 清除遗留数据，防止上一次分配的脏数据干扰类型判断或导致后续 ARC 释放崩溃。
    - **性能收益**：极大减少了自举编译器（及所有玄铁程序）在运行时因为频繁创建、销毁 AST 节点和临时变量带来的内核态切换开销，初步测试内存分配速度有显著提升。

## v0.14.2 (2026-04-17)
- **自举编译器 Bug 修复**：
    - **ARC 内存安全**：修复了 LLVM 后端在处理方法参数和表达式返回值时的双重释放（double free）与内存泄漏问题。调整了 `compileExpression` 返回 +1 引用的约定，并在 return 语句和方法局部作用域退出时进行了正确的引用计数管理。
    - **实例内存分配**：在 `xt_runtime.c` 的 `xt_instance_new` 中，将实例字段的内存分配从 `malloc` 改为 `calloc`，确保未初始化的字段在 ARC 释放时不会因为野指针导致段错误（0xC0000005）。
    - **内置类型方法调用规范**：修正了在自举源码（`编译.xt`）中对 `XTDict`（基本类型）错误使用 `.键()` 方法导致编译后程序解引用空指针（XT_NULL）崩溃的问题。玄铁中的字典应直接使用 `遍历 键 于 字典` 进行遍历。
