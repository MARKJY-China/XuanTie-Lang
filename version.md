玄铁版本（XuanTie SemVer）框架：
版本 SemVer：主版本.次版本.修订号.测试号

主版本（Major）：破坏性变更/产品里程碑
次版本（Minor）：向后兼容功能/部分新功能/大更新
修订号：向后兼容缺陷修复/Bug 修复/小更新

## 0.9.3 (2026-04-06)
- **容器权能进化：工业级数组操作**：
    - **数组高级方法**：新增 `追加` (Append)、`截取` (Slice)、`删` (Remove)、`插` (Insert)、`找` (IndexOf) 成员方法。
    - **高阶函数支持**：新增 `映射` (Map) 和 `过滤` (Filter) 方法，支持函数式编程风格。
    - **内存优化**：优化了数组切片操作的底层实现，减少不必要的内存拷贝。

## 0.9.2 (2026-04-06 2:45)
- **底层权能深耕：字节处理与命令行支持**：
    - **字节数据类型 `字节` (Bytes)**：引入原生字节切片支持，通过 `字节.从字` 和 `字节.到字` 实现与文本的互转。支持 `长度` 属性和下标索引。
    - **命令行参数 `系统.参数`**：内置支持获取程序启动参数，极大增强了自动化工具和编译器的编写权能。
    - **统一运行时行为**：优化了编译器与解释器在 `&` 拼接和 `inspect` 输出上的行为一致性。
    - **常用容器增强**：为 `数组` (Array) 增加了 `.长度` 属性。

## 0.9.1 (2026-04-06 1:45)
- **底层互操作性与跨平台增强**：
    - **外部函数接口 `外` (FFI)**：引入内置库 `外`，支持动态加载 `.dll` (Windows) 或 `.so` (Linux) 库并调用其中的 C 函数。
    - **交叉编译支持**：升级命令行工具，支持 `--pt` （平台）(GOOS) 和 `--jg` （架构）(GOARCH) 参数，方便在 Windows 下直接构建 Linux 生产环境产物。
    - **字节码与底层优化**：为自举做准备，优化了底层字符串与字节流的转换效率。

## 0.9.0 (2026-04-05 17:37)
- **网络与系统权能元年**：
    - **网络原语 `连`、`听`、`求`**：
        - `求` (Fetch)：一行代码获取网页内容，完美集成 `结果` 容器链式调用。
        - `听` (Listen)：内置 TCP 服务端监听，默认非阻塞后台运行。
        - `连` (Connect)：内置 TCP 客户端连接，返回可读写的 `流` (Stream) 对象。
    - **系统原语 `执`、`道`**：
        - `执` (Execute)：支持调用操作系统命令并获取输出结果。
        - `道` (Channel)：引入原生通道支持，实现异步任务间的安全数据通讯。
    - **标准库单字化**：新增 `时` (Time) 对象，支持 `睡` (Sleep) 和 `现` (Now) 方法。
    - **链式调用增强**：优化了 `结果.接着` 和 `结果.否则` 的逻辑，现在支持无限链式调用。

## 0.8.0 (2026-04-05 4:24)
- **面向对象进阶：继承与保护**：
    - **继承支持 `承`**：引入关键字 `承` (Inherit)，支持类与类之间的单继承。子类自动获得父类的所有非私有属性和方法。
    - **受保护访问 `护`**：正式激活 `护` (Protected) 修饰符。标记为 `护` 的成员允许在类及其子类内部访问，但禁止外部代码直接访问。
    - **继承链查找**：优化了 Evaluator 和 Compiler 的查找算法，支持在完整的继承链中动态解析成员。
    - **文档升级**：在语法指南中增加了详细的继承与权限控制讲解。

## 0.7.2 (2026-04-05)
- **原生方法与全能构造器**：
    - **具名函数支持**：现在支持使用 `函 名字(参数) { ... }` 定义全局具名函数或类方法。
    - **构造函数 `造`**：在 `型` 内部定义 `函 造` 即可创建构造函数。实例化时支持 `造 类型(参数)` 调用构造逻辑。
    - **权限修饰符扩展**：新增 `公` (Public) 和 `护` (Protected) 关键字。配合 `私` (Private) 构成了完整的可见性控制体系。
    - **双模实例化**：支持 `造 类型(...)`（构造函模式）和 `造 类型{...}`（字面量模式）共存。

## 0.7.1 (2026-04-05 1:25)
- **模型权能增强与安全约束**：
    - **成员赋值支持**：现在支持通过 `对象.成员 = 新值` 的语法直接修改实例属性。
    - **私有属性标记**：引入关键字 `私` (Private)。在 `型` 定义中使用 `私 设` 或 `私 常` 标记的属性，仅能被该类的内部方法访问和修改，外部访问将触发运行时错误。
    - **方法作用域同步**：修正了实例方法内部对同名属性赋值时，无法同步更新到实例状态的问题。

        - 支持默认字段初始化和实例化时的数据覆盖。
        - 实现了基础的成员访问 (`obj.member`) 和方法绑定逻辑。
    - **编译器增强**：Go 编译器同步支持了 JSON 序列化和类定义逻辑。

## 0.6.0 (2026-04-04)

    - **兼容性**：保留了原有的双字关键字作为别名，确保平滑过渡。

## 0.5.0 (2026-04-04)
- **语法语义重大变革**：
    - **条件控制关键字重构**：为了追求更高的“高熵信息含量”与文学性，将原有的 `如果`、`否则` 关键字全面重构为 `若` (If) 与 `否` (Else)。

## 0.4.1 (2026-04-04)
- **依赖管理安全性增强**：
    - **循环引用检测**：解释器和转译器现在均具备循环引用（Circular Dependency）检测能力。当 A 引用 B 且 B 间接或直接引用 A 时，系统将立即报错并输出完整的引用链（Import Trace），防止进入无限递归导致的堆栈溢出。
    - **编译错误提示优化**：改进了转译器的错误处理流，转译过程中的逻辑错误（如循环引用）现在会以友好的格式在终端输出，而不再触发 Go 的 panic。

## 0.4.0 (2026-04-04)
- **引用逻辑语义优化**：
    - **上下文感知引用**：实现了 `引用` 的上下文感知。当 `引用` 用于变量赋值（如 `变量 x = 引用 "..."`）时，它被视为“模块加载”，此时会自动抑制被引用文件中的顶层副作用（如 `打印`、`如果`、`当` 等逻辑块），仅导出变量和函数定义。
    - **直接执行模式**：当 `引用` 独立作为语句存在时，保持“脚本执行”模式，完整运行被引用文件的所有内容。
    - **全平台一致性**：确保了解释器（Evaluator）与转译器（Compiler）在处理该语义时表现一致。

## 0.3.9 (2026-04-04)
- **转译器 (Transpiler) 全面增强**：
    - **模块引用支持**：实现了递归的 `引用` (Import) 转译。现在被引用的模块会自动被转译为 Go 函数并内联到最终的二进制文件中，支持完整的模块导出逻辑。
    - **高级特性支持**：新增对 `Result` 类型、链式调用 (`接着`/`否则`)、`异步` (Async)、`并行` (Parallel) 以及 `等待` (Await) 的转译支持。
    - **混合运算优化**：改进了运行时算术函数，现在完美支持 `整数` 与 `小数` 的混合加减乘除运算。
    - **异常处理完善**：实现了 `尝试/捕捉` (Try/Catch) 的转译，现在能捕获如“除零错误”等运行时异常。

## 0.3.8 (2026-04-04)

## 0.3.7 (2026-04-04)
