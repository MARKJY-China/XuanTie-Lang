# 玄铁 (XuanTie) 语言指南 v0.8.0

玄铁是一门旨在融合“高熵信息含量”与“原生异步并发”特性的中文编程语言，其语法极其精简，但同时因每个字符携带的信息量更大，同时视觉区分度更高，理论上能减少长时间编码的认知负担和错误率。它既能让 AI 更直观地理解逻辑因果，也为中文开发者提供了极高的表达效率和可读性。

## 1. 基础语法

### 变量与常量
```xuantie
设 名字 = "玄铁"
常 版本 = 0.6
```

### 示出 (Show)
```xuantie
示("你好，" & 名字)  // 使用 & 进行字符串拼接
```

## 2. 数据类型
- **基础**: `整` (123), `小数` (3.14), `字` ("abc"), `逻` (真/假), `空` (空)
- **容器**: `数组` ([1, 2, 3]), `字典` ({"键": "值"})
- **进阶**: `结果` (Result), `任务` (Task)

### 索引访问
支持通过 `[]` 访问数组、字典和字符串的元素。
```xuantie
设 文本 = "玄铁"
示(文本[0]) // 输出: 玄

设 列表 = [1, 2, 3]
示(列表[1]) // 输出: 2

设 映射 = {"名": "玄铁"}
示(映射["名"]) // 输出: 玄铁
```

### 逻辑运算
支持标准的逻辑运算，并具备**短路求值**特性。
- `且` (AND): 两个条件都为真时结果为真。
- `或` (OR): 其中一个条件为真时结果为真。
- `非` (NOT): 对逻辑值取反。
- `是` (IS): 用于判断类型或逻辑相等。
- `-` (负号): 对数值取反。

```xuantie
若 (1 > 0) 且 (2 > 1) { 示("真") }
设 结果 = 非 真 // 结果为假
若 1 等于 1 { 示("一等于一") }
若 1 是 整 { 示("1 是整数") }
```

## 3. 流程控制

### 条件判断 (若-抑-否)
玄铁采用极其简练的条件控制语法。
> `抑`古义为“或者”。在这里，它表示“但若条件为真则执行”。
```xuantie
若 10 > 5 {
    示("正确")
} 抑 10 == 5 {
    示("等于")
} 否 {
    示("错误")
}
```

### 循环与遍历
玄铁支持条件循环以及针对容器的快速遍历。

**当 (While) 循环**
```xuantie
设 i = 0
当 i < 3 {
    示(i)
    i = i + 1
}
```

**循 (Loop) 无限循环**
```xuantie
设 k = 0
循 {
    k = k + 1
    若 k > 10 { 断 }
    //如果k大于10那么断开（跳出）循环
}
```

**遍历 (For-In) 循环**
支持遍历数组、字典（键）以及字符串（字符）。
```xuantie
设 列表 = [10, 20, 30]
遍历 元素 于 列表 {
    示(元素)
}

设 映射 = {"名": "玄铁", "版": 0.6}
遍历 键 于 映射 {
    示(键 & ": " & 映射[键])
}
```

**循环控制**
- `断` (Break): 立即终止并跳出当前最内层循环。
- `续` (Continue): 跳过当前循环体剩余代码，直接进入下一次迭代。
```xuantie
遍历 i 于 [1, 2, 3, 4, 5] {
    若 i == 2 { 续 } // 跳过 2
    若 i == 4 { 断 } // 到 4 停止
    示(i)
}
```

## 4. 函数
```xuantie
设 加法 = 函(a, b) {
    返 a + b
}
设 结果 = 加法(1, 2)
```

## 5. 进阶特性 (v0.3.0+)

### JSON 原生支持 (v0.7.0)
玄铁内置了极简的序列化与反序列化工具。
- **`化` (Serialize)**：将对象转化为 JSON 字符串。
- **`解` (Deserialize)**：将 JSON 字符串还原为玄铁对象（字典或数组）。

```xuantie
设 数据 = {"名": "玄铁", "值": 100}
设 文本 = 化(数据)  // 得到 JSON 字符串
设 对象 = 解(文本)  // 还原为字典
示(对象["名"])
```

### 面向对象初步 (v0.7.0+)
玄铁提供了轻量级的类与对象支持。
- **`型` (Type/Class)**：定义一个数据模型。
- **`造` (New/Construct)**：基于模型创建实例或定义构造函数。
- **`承` (Inherit)** (v0.8.0)：实现类之间的继承关系，子类可复用父类的属性和方法。
- **权限修饰符** (v0.7.2+)：
    - **`公` (Public)**：默认级别，外部可自由读写。
    - **`私` (Private)**：仅限本类内部访问，外部和子类均不可见。
    - **`护` (Protected)** (v0.8.0)：受保护级别，允许本类和子类访问，禁止外部直接访问。

#### 1. 基础定义与构造
```xuantie
型 坐标 {
    设 x = 0
    私 设 y = 0
    
    // 构造函数
    函 造(初始x, 初始y) {
        x = 初始x
        y = 初始y
    }
}
设 点 = 造 坐标(10, 20)
```

#### 2. 继承 (承) 与 保护 (护)
**`承`** 实现了代码复用，而 **`护`** 确保了敏感逻辑在“家族链”内部的安全流动。

```xuantie
型 账户 {
    公 设 户名 = "访客"
    护 设 余额 = 0  // 只有家族成员（子类）能动余额
    私 设 内部标记 = "SECRET" // 只有账户类自己能看
}

型 信用卡 承 账户 {  // 信用卡 继承了 账户
    函 存款(数额) {
        余额 = 余额 + 数额 // 合法：子类可以访问“护”属性
        示("存入成功，当前余额: " & 余额)
    }
}

设 我的卡 = 造 信用卡()
我的卡.存款(1000)
示(我的卡.户名)   // 合法：公有属性
// 示(我的卡.余额) // 报错：外部禁止访问受保护属性
```

### 网络与系统原语 (v0.9.0)
玄铁通过极简的单字原语，将复杂的网络通讯和系统交互降维到“管道读写”级别。

#### 1. 网络请求 (`求`)
一行代码获取互联网数据，原生返回 `结果` 容器。
```xuantie
求("https://api.github.com").接着(函(内容) {
    示("收到数据: " & 内容)
})
```

#### 2. Socket 通讯 (`听` 与 `连`)
支持原生的 TCP 流通讯。
- **`听` (Listen)**：开启服务端监听，默认非阻塞后台运行。
- **`连` (Connect)**：建立客户端连接，返回 `结果(流)`。
- **`流` (Stream)**：通过 `读`、`写`、`关` 操作通讯。

```xuantie
// 服务端
听(9000, 函(管道) {
    设 数据 = 管道.读()
    管道.写("已收到: " & 数据)
    管道.关()
})

// 客户端
连("127.0.0.1:9000").接着(函(管道) {
    管道.写("你好")
    管道.关()
})
```

#### 3. 系统执行 (`执`)
调用操作系统命令，并获取其输出。
```xuantie
执("git status").接着(函(输出) {
    示(输出)
})
```

#### 4. 通道通讯 (`道`)
在不同的异步任务（协程）之间安全地传递数据。
```xuantie
设 消息道 = 道
异步 { 消息道.送("秘密数据") }
设 收到 = 消息道.收()
```

### 结果处理 (Result)
引入函数式错误处理，避免嵌套的 `if err != nil`。
> 这种写法主要参照了 JavaScript (Promises) 、 Rust ( `Result<T, E>` )
```xuantie
设 获取数据 = 函() { 返 成功("数据") }

获取数据()
    .接着(函(内容) { 示("成功: " & 内容) })
    .否则(函(错误) { 示("失败: " & 错误) })
```
相比传统，如C语言的示例写法：
```
// C 语言风格：不断地检查返回状态
int err = get_data(&data);
if (err == SUCCESS) {
    int err2 = process_data(data, &result);
    if (err2 == SUCCESS) {
        save(result);
    } else {
        handle_error(err2);
    }
} else {
    handle_error(err);
}
```
这种写法会导致著名的“金字塔嵌套”或大量的 if (err != 0) goto cleanup 。
玄铁的解决方案是“管道化”：
 - `.接着`的本质： `获取数据()` 返回的不是一个简单的整数或指针，而是一个 结果 (Result) 容器。这个容器像一个包裹，里面可能装着“成功的数据”，也可能装着“失败的错误”。
 - 链式调用 ： `.接着` 是这个容器的一个方法。它的逻辑是：“如果你（容器）里面是成功的，就请执行我传给你的函数；如果你是失败的，请跳过我，把错误直接传给下一个人。”
 - 换行与点 ：这是一种 视觉对齐 。它让代码读起来像一个 流水线清单 ：
  1. 获取数据
  2. `.接着` 处理它
  3. `.接着` 转换它
  4. `.否则` 报错

### 异步与并行 (Async/Parallel)
原生支持并发，底层基于 Go 协程。
```xuantie
// 异步执行
设 任务 = 异步 { 耗时任务() }
设 结果 = 等待(任务)

// 并行执行多个块
设 结果集 = 并行 {
    { 任务1() },
    { 任务2() }
}
```

### 异常捕获 (Try/Catch)
捕获运行时错误（如除零、未定义变量）。
```xuantie
尝试 {
    10 / 0
} 捕捉 (错误) {
    示("捕获异常: " & 错误)
}
```

## 7. 内置标准库 (v0.3.1+)
玄铁开始构建其 IO 与网络能力。

### 文件 IO (v0.6.0)
- `文件.开(路径)`: 打开文件。
- `文件.关(句柄)`: 关闭文件。
- `文件.读(路径)`: 返回 `结果`，包含文件字符串内容。
- `文件.写(路径, 内容)`: 返回 `结果`，布尔值表示是否成功。
- `文件.添(路径, 内容)`: 追加内容。
- `文件.删(路径)`: 删除文件。
- `文件.建(路径)`: 创建文件。
- `文件.拷(源, 目标)`: 复制文件。
- `文件.移(源, 目标)`: 移动文件。

### 网络
- `网络.获取(URL)`: 返回 `结果`，包含网页正文内容。支持配合 `异步` 使用。

### 字符串处理
玄铁提供了强大的字符串处理内置函数。
- `字符串.长度(s)`: 获取字节长度。也可直接使用 `s.长度`。
- `字符串.子串(s, 起始, 长度)`: 截取子字符串。
- `字符串.替换(s, 旧, 新)`: 全量替换子串。
- `字符串.分割(s, 分隔符)`: 返回分割后的数组。

### 数学运算
- `数学.绝对值(n)`: 获取绝对值。
- `数学.正弦(n)`, `数学.余弦(n)`: 三角函数。
- `数学.平方根(n)`: 计算平方根。
- `数学.随机数()`: 返回 0.0-1.0 之间的浮点数。
- `数学.随机数(n)`: 返回 0-(n-1) 之间的整数。

### 时间处理
- `时间.现在()`: 获取当前 Unix 时间戳。
- `时间.休眠(ms)`: 程序暂停指定的毫秒数。
- `时间.格式化(ts, 格式)`: 将时间戳格式化为字符串。格式遵循 Go 的 layout (如 `2006-01-02 15:04:05`)。

### 模块化 (v0.3.2)
使用 `引` 关键字加载其他脚本，并获得其导出的变量。
- **路径解析**：相对路径（如 `./lib.xt`）是相对于**当前代码文件**所在的目录。这使得你可以轻松地将模块组织到不同的子目录中。
```xuantie
设 工具 = 引 "./utils.xt"
示(工具.加法(1, 2))
```

### 编译为二进制 (v0.3.7+)
玄铁支持将 `.xt` 脚本直接编译为原生的 `.exe` (Windows) 或二进制文件。
```bash
xuantie build 你的程序.xt
```
- **优势**：执行速度大幅提升，不再依赖玄铁解释器，且源码已转换为机器码。
