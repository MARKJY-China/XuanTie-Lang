玄铁是一门旨在融合“高熵信息含量”与“原生异步并发”特性的中文编程语言。它既能让 AI 更直观地理解逻辑因果，也为中文开发者提供了极高的表达效率。

## 1. 基础语法

### 变量与常量
```java
设 名字 = "玄铁"
常 版本 = 0.16
```

### 字符串插值 (String Interpolation)
玄铁支持在字符串中使用 `#{表达式}` 进行变量或逻辑插值，这比使用 `&` 拼接更加简洁。

```java
设 人 = "开发者"
示("你好，#{人}！现在是 #{时.现()} 毫秒")
```

### 类型转换
玄铁提供内置函数进行类型强制转换：

+ `整(x)`: 转换为整数。如果是字符串且格式不对，返回 0 或引发异常。
+ `小数(x)`: 转换为小数。
+ `字(x)`: 转换为字符串。

```java
设 s = "123"
设 n = 整(s) // 123
设 f = 小数(n) // 123.0
```

### 示出与输入
```java
示("你好，" & 名字)  // 使用 & 进行字符串拼接
设 回复 = 输入("请输入：") // 获取用户输入
```

### 结果容器 (Result)
`结果` 类型是玄铁错误处理的核心。通过内置函数 `成功(值)` 或 `失败(错误)` 创建。

```java
设 最终值 = 成功(100)
若 最终值.成功 {
    示("结果是: #{最终值.值}")
}
```

## 2. 数据类型

+ **基础**: `整` (123), `小数` (3.14), `字` ("abc"), `逻` (真/假), `空` (空)
+ **容器**: `数组` ([1, 2, 3]), `字典` ({"键": "值"})
+ **进阶**: `任务` (Task), `字节` (Bytes), `道` (Channel)

### Arena 内存管理
针对短生命周期高频分配场景（如编译器），支持使用 Arena 内存池一键回收。

```rust
外 函 xt_arena_new(大小: 整): 整
外 函 xt_arena_use(池: 整)
外 函 xt_arena_destroy(池: 整)

设 池 = xt_arena_new(1024 * 1024)
xt_arena_use(池) // 后续所有分配均在池中进行
// ... 执行逻辑 ...
xt_arena_destroy(池) // 一键回收所有内存
```

## 3. 流程控制

### 条件判断 (若-抑-否)
```java
若 10 > 5 {
    示("正确")
} 抑 10 == 5 {
    示("等于")
} 否 {
    示("错误")
}
```

### 循环控制
**当 (While) 循环**
```java
设 i = 0
当 i < 3 {
    示(i)
    i = i + 1
}
```

**遍历 (For-In) 循环**
```java
遍历 元素 于 [1, 2, 3] {
    示(元素)
}
```

## 4. 网络与系统原语 (v0.9.0+)

玄铁通过极简的单字原语处理复杂的 IO 任务。

### 1. 网络请求 (`求`)
一行代码获取互联网数据，原生返回 `结果` 容器。
```rust
求("https://api.github.com").接着(函(内容) {
    示("收到数据长度: #{内容.长度}")
})
```

### 2. 服务监听 (`听`)
`听` 关键字具有多态性，能自动处理 TCP 或 HTTP 协议。
```rust
// 开启 TCP 监听
听(9000, 函(流) {
    设 数据 = 流.读()
    流.写("已收到: " & 数据)
    流.关()
})
```

### 3. 并发通道 (`道`)
用于在不同任务之间传递数据，实现 CSP 模型。
```rust
设 通 = 道(10) // 创建容量为 10 的通道
异步 {
    通.发("来自后台的消息")
}
设 消息 = 通.收()
示(消息)
```

### 4. 系统执行 (`执`)
调用操作系统命令。
```rust
执("dir").接着(函(输出) {
    示("当前目录列表: #{输出}")
})
```

## 5. 异常捕获 (Try/Catch)

玄铁支持结构化的异常处理。

```rust
尝试 {
    设 结果 = 10 / 0
} 捕捉 (错) {
    示("捕获到错误!")
    // '错' 对象包含详细的错误信息
}
```

## 6. 内置标准库方法

### 字符串 (字)
+ `.长度`: 获取字节长度。
+ `.截取(始, [终])`: 截取子串。
+ `.包含?(子串)`: 是否包含特定内容。
+ `.索引(子串)`: 查找位置。

### 数组
+ `.长度` / `.大小`: 获取元素个数。
+ `.追加(值)`: 在末尾添加。
+ `.连接(分隔符)`: 将元素拼接为字符串。

### 字典
+ `.大小`: 键值对数量。
+ `.含?(键)`: 检查是否存在。

### 文件
+ `文件.读(路径)`: 返回 `结果`，包含文件内容。
+ `文件.写(路径, 内容)`: 返回 `结果`，表示是否成功。

## 11. 玄铁的核心优势

与 Python、Java、Go、Rust、JavaScript 这些主流语言对比，**玄铁**（XuanTie）的核心优势可以概括为一句话：

> **"高熵语义下的逻辑降维"**

玄铁不是在"重造轮子"，而是在"简化轮子的驾驶舱"。它是给那些追求极致效率、厌倦了样板代码、希望代码逻辑一眼到底的开发者准备的——尤其是中文母语者。

---

### 1. 语义熵的降维：单字即权能

在主流语言中，网络、系统、并发操作通常被封装在"重型库"中，需要大量样板代码才能启动。

+ **Java/Python**：你需要 `import` 库，实例化 `Client`，配置 `SocketFactory`，处理各种生命周期回调。
+ **玄铁**：将这些能力直接提炼为**单字原语**，内置进语言骨架。

| 功能 | 玄铁 | Python/JS | 说明 |
| --- | --- | --- | --- |
| 网络请求 | `求(URL)` | `requests.get(url)` / `fetch(url)` | 向外界索取 |
| 服务器监听 | `听(端口, 函)` | 复杂的 server 配置 | 等待回应 |
| TCP 连接 | `连(地址)` | socket 多步初始化 | 建立连接 |
| 系统命令 | `执("git status")` | subprocess.run | 执行 |
| 并发通道 | `道` | Queue/Channel 库 | 消息往来 |
| 异步任务 | `异步 { ... }` | `async def` + `asyncio` | 回头再做 |

**优势**：在玄铁中，网络与并发不再是"外部库"，而是语言的**本能**。代码行数通常能缩减到主流语言的 1/3，且语义更直观。一行 `求("https://api.github.com").接着(函(数据) { ... })` 就能完成一次完整的网络调用。

---

### 2. 错误处理的降维：从"条件判断"到"管道流动"

**这是玄铁相比 Go 和 C 最大的优势。**

+ **Go 语言**：著名的 `if err != nil` 占据 30% 以上的代码量，逻辑被无数判断切碎。
+ **C 语言**：需要层层 `if (err == SUCCESS)` 嵌套，形成"金字塔"或大量 `goto cleanup`。
+ **玄铁**：采用**结果容器化**方案，配合完整的**函数式组合算子**。

玄铁的 `结果` (Result) 容器在 v0.12 之后拥有完整的函数式操作能力：

```java
设 最终值 = 整("123")              // 字符串转整数，返回结果容器
    .映射(函(x) { 返 x * 2 })     // 成功时转换
    .复原(函(e) { 返 0 })          // 失败时恢复
    .或(-1)                        // 最终兜底
```

一条链式调用可以串联多个可能失败的操作，错误会自动在管道中"跳跃"直到被捕获，**主干逻辑（Happy Path）清晰可见**。

配合**结构化异常**（v0.13+），运行时错误不再是"字符串消息"，而是带 `.消息` 和 `.堆栈` 的对象：

```java
尝试 {
    危险操作()
} 捕捉 (错) {
    示("出错了: " & 错.消息)
    示("调用栈: " & 错.堆栈)
}
```

**优势**：玄铁同时提供了两种错误处理哲学——链式容器（Result）用于业务逻辑，异常捕获（Try/Catch）用于底层意外。一个语言，两套武器，按场景自由切换。

---

### 3. 并发模型的降维：从"工具类"到"空间语法"

相比 JavaScript、Python、Java 的并发模型：

+ **主流语言**：开发者思考逻辑时用中文，写代码时要翻译成英文词汇（`asynchronous`、`callback`、`instantiate`、`goroutine`）。这中间存在一次"**脑内转译**"，是低级逻辑错误的主要来源。
+ **玄铁**：逻辑词汇与思维词汇**完全对齐**，没有转译损耗。

| 思维 | 玄铁语法 | 主流语言 |
| --- | --- | --- |
| 回头再做 | `异步 { ... }` | `async def` / `go func() {}` |
| 等它做完 | `等待(任务)` | `await` / `task.join()` |
| 同时做几件事 | `并行 { ... }` | `Promise.all` / `sync.WaitGroup` |
| 消息往来的道路 | `道` | `channel` / `Queue` |

**优势**：对中文母语者，玄铁代码几乎没有"阅读阻力"。这种**低认知负荷**在处理复杂网络协议和多线程逻辑时，能显著降低低级逻辑错误的发生率。底层基于 Go 协程的 M:N 调度，支持成千上万并发任务，性能对标 Go。

### 4. 内存管理的降维：四层架构，按需控制

玄铁设计了业内独特的**四层内存管理架构**，让开发者从"全自动"到"手动"按需选择控制粒度：

```
编译期：
  ├─ 环检测+自动降级（处理循环引用，零运行时开销）
  └─ Arena 自动推导（生命周期明确的对象自动区域分配）

运行时：
  ├─ ARC 原子引用计数（动态对象管理，无 GC 停顿）
  └─ 弱引用兜底（手动或自动标注 `弱 设`）
```

**面向三种开发者的控制粒度：**

+ **普通开发者**：默认 ARC + 编译期 Arena 推导，完全无感知
+ **性能敏感开发者**：显式声明 Arena 区域，批量释放
+ **底层开发者**：通过 FFI 直接调用 `xt_arena_new/use/destroy`，像 C 一样手动控制

```java
// 手动开启 Arena 内存池（适合编译器、游戏单帧逻辑、HTTP 请求处理）
外 函 xt_arena_new(大小: 整): 整
外 函 xt_arena_use(池: 整)
外 函 xt_arena_destroy(池: 整)

设 池 = xt_arena_new(1024 * 1024)
xt_arena_use(池)   // 后续所有分配均在池中进行
// ... 高频短命对象操作 ...
xt_arena_destroy(池)  // 一键回收，零碎片
```

**优势**：
- 无 GC 停顿，性能可预测（对标 Swift、Rust）
- 循环引用自动处理（Python 的 ARC 做不到，Swift 需要手动 weak）
- 渲染引擎/游戏/编译器内部可手动榨取极致性能，应用层依然全自动安全
- 将来支持**编译期内存预算**，嵌入式场景可在编译期静态验证内存上限

这套架构在现有主流语言中独一份——**编译期环检测 + Arena 推导 + ARC + 弱引用**的组合，是玄铁的原创设计。

### 5. 表达力的降维：从"写代码"到"描述逻辑"

玄铁吸收了现代语言的所有核心特性，并用中文语义重新表达：

+ **字符串插值** (`"#{表达式}"`)：告别 `+ str(x) +` 的拼接地狱
+ **模式匹配** (`匹配`)：值匹配、类型匹配、通配符 `_` 一次解决分支爆炸
+ **泛型与约束** (`<T 承 接口>`)：编译期类型安全
+ **接口契约** (`口`)：鸭子类型+显式契约
+ **谓词命名** (`空?`/`存在?`/`包含?`)：受 Ruby/Clojure 启发，用 `?` 结尾表达"判断型函数"
+ **运算符重载**（`_加_`/`_等_`）：让自定义类也能参与 `+`、`==` 运算
+ **访问控制** (`公`/`护`/`私`)：完整的封装体系
+ **原生 JSON** (`化`/`解`)：一步序列化/反序列化
+ **文档注释** (`///`)：自动挂载到函数对象，支持工具链提取

配合**原生测试框架**（v0.13+），工程化能力开箱即用：

```java
/// 计算两数之和
/// 这是一个带文档注释的函数
函 加(a, b) { 返 a + b }

测试 "加法基础校验" {
    系统.断言(加(1, 2) == 3, "1+2 应该等于 3")
    系统.断言(加(0, 0) == 0)
}
```

**优势**：这些特性单独拿出来，任何现代语言都有。但**同时拥有**、并且用**统一的中文语义表达**，这是玄铁独一份的体验。

### 6. "你这关键字简化的那么极致，跟文言文似的，我们哪看得懂？"

> "英文编程是在用'碎片'拼凑逻辑，而玄铁是在用'内核'驱动逻辑。你看得惯 `int` 和 `var`，就没有理由看不懂 `整` 和 `设`。这不叫文言文，这叫'高熵编程'。"

#### 英文编程本质上就是极致简化的英文缩写

+ `int` 是 integer 的缩写
+ `var` 是 variable 的缩写
+ `func`/`fn` 是 function 的缩写
+ `ls` 是 list 的缩写

西文开发者为了输入效率，**不得不**把单词切碎。

**而中文的优势在于**：一个字就是一个完整的语义单位。我们不需要缩写，我们只需要"提炼"。

#### "视觉熵"与"逻辑骨架"

玄铁追求单字化，是为了让**逻辑骨架一眼到底**。

当关键字全是单字（`设`、`若`、`听`、`求`）时，它们在代码中像逻辑的"坐标点"。由于它们足够短，**业务变量名和逻辑结构变得非常突出**。

这就像看一张地图——如果路标占了一半面积，你就看不清路了。玄铁把路标缩小，把"路"留给开发者。

#### "文言文"其实是褒义词

说玄铁像文言文，其实是因为**文言文是中文信息密度最高的形式**。

+ 当你看到 `求`，大脑直觉触发"向外界索取"，这比 `HttpClient.executeRequest` 触发语义快得多，而且输入只需要 `qiu` 三个字符。
+ 当你看到 `听`，大脑直觉触发"聆听/监听/等待回应"，这比 `ServerSocket.accept` 更符合人类本能。

**编程不是在写作文，而是在构建逻辑。** 最顶级的逻辑，往往用最精炼的词汇表达最复杂的动作。

### 7. AI 时代的必然选择

在 AI 辅助编程成为主流的今天，**冗长的样板代码是效率的最大杀手**。

玄铁的单字原语，实际上是为 AI 和高阶开发者设计的"**语义降维**"：

+ AI 能更精准地理解 `求` 的意图，不需要在 `requests.get`、`urllib.urlopen`、`http.client` 之间做选择
+ 开发者可以在**更短的行数里审视更复杂的并发逻辑**
+ Prompt 可以直接使用中文思维描述需求，AI 生成的代码无需"翻译"回中文理解

正如 C 语言简化了汇编，Python 简化了 C，**玄铁是在简化现代编程中日益臃肿的"语义冗余"**——这是 AI 时代对编程语言的必然要求。

### 8. 工程级底层，不是玩具

玄铁不是"把关键字换成中文"的皮肤级产物，它的底层架构对标工业级语言：

| 维度 | 玄铁的实现 |
| --- | --- |
| 前端 | 自举编译器（玄铁自写）+ 种子编译器（Go） |
| 中端 | LLVM IR |
| 后端 | C 运行时（Tagged Pointers + 原子 ARC + 内存池） |
| 目标 | 原生机器码，跨平台（Windows/Linux/macOS/WASM） |
| 类型系统 | 泛型、约束、接口、联合、空安全 |
| 并发 | Go 协程（当前）→ 工作窃取调度器（v1.0+） |
| FFI | 原生 C 动态库调用，自动 UTF-16 适配 |
| 内存 | 四层架构：环检测 + Arena + ARC + 弱引用 |
| 工具链 | 原生测试、文档注释、彩色错误、交叉编译 |

v0.15.6 阶段，自举编译器已**图灵完备**，距离 v1.0.0 的自举闭环只差最后一步。
