# 玄铁 (XuanTie) 语言指南 v0.9.1

玄铁是一门旨在融合“高熵信息含量”与“原生异步并发”特性的中文编程语言，其语法极其精简，但同时因每个字符携带的信息量更大，同时视觉区分度更高，理论上能减少长时间编码的认知负担和错误率。它既能让 AI 更直观地理解逻辑因果，也为中文开发者提供了极高的表达效率和可读性。

## 1. 基础语法

### 变量与常量
```xuantie
设 名字 = "玄铁"
常 版本 = 0.6
```

### 字符串插值 (String Interpolation)
玄铁支持在字符串中使用 `${表达式}` 进行变量或逻辑插值，这比使用 `&` 拼接更加简洁。
```xuantie
设 人 = "开发者"
示("你好，${人}！现在是 ${时.现()} 毫秒")
```

### 类型转换
玄铁提供内置函数进行类型强制转换：
- `整(x)`: 转换为整数。如果是字符串且格式不对，返回 `结果`。
- `小数(x)`: 转换为小数。
- `字(x)`: 转换为字符串。

```xuantie
设 s = "123"
设 n = 整(s).值 // 123
设 f = 小数(n) // 123.0
```

### 示出与输入
```xuantie
示("你好，" & 名字)  // 使用 & 进行字符串拼接
设 回复 = 输("请输入：") // 获取用户输入
```

## 2. 数据类型
- **基础**: `整` (123), `小数` (3.14), `字` ("abc"), `逻` (真/假), `空` (空)
- **容器**: `数组` ([1, 2, 3]), `字典` ({"键": "值"})
- **结果**: `结果` (Result) —— 用于错误处理，通过内置函数 `成功(值)` 或 `失败(错误)` 创建。支持 `.成功`、`.值`、`.错误` 属性访问，以及 `.接着(函)` 和 `.否则(函)` 链式调用。
- **进阶**: `任务` (Task), `字节` (Bytes)

### 类型推导与 FFI 增强 (v0.10.0)
玄铁进一步简化了类型系统的使用，并增强了底层交互能力。

- **自动类型推导**: 声明变量时可省略类型注解，编译器将根据初始值自动推断。
- **泛型参数推导**: 实例化泛型类或调用泛型函数时，若参数类型明确，可省略 `<T>`。
- **FFI 原型定义**: 支持为外部动态库函数定义参数和返回类型。

```xuantie
// 1. 变量推导
设 数字 = 100 // 自动识别为整

// 2. 泛型推导
型 盒子<T> { 设 内容: T; 函 造(物: T) { 内容 = 物 } }
设 盒 = 造 盒子(123) // 自动推导为 盒子<整>

// 3. FFI 增强
设 库 = 外.加载("user32.dll").值
设 弹窗 = 库.函数("MessageBoxW", [整, 字, 字, 整], 整)
// 弹窗(0, "内容", "标题", 0)
```

### 模块化命名空间 (v0.9.9)
玄铁支持模块化开发，允许通过 `引` 关键字引入外部文件。

- **显式命名空间**: `引 "路径" 予 别名`。
- **可见性控制**: 模块内部变量默认私有，使用 `公` 关键字明确导出的成员。

```xuantie
// lib.xt
公 设 名字 = "玄铁库"
设 秘密 = "私有变量" // 外部不可见

公 函 招呼(人) {
    示("你好，" & 人)
}

// main.xt
引 "./lib.xt" 予 库
示(库.名字)
库.招呼("世界")
// 库.秘密 // 报错：不支持的成员调用
```

### 类型注解 (v0.9.8)
玄铁支持**可选类型注解**，允许在声明变量、定义函数参数或指定函数返回值时，明确其预期的类型。

- **基础注解**: `设 变量: 类型 = 值`
- **联合类型 (Union)**: 使用 `|` 表示多种可能类型，如 `设 值: 字 | 整 = 123`。
- **空安全 (Nullable)**: 使用 `?` 表示可为空，如 `设 名字: 字? = 空`。未加 `?` 的类型默认不允许赋值为 `空`。
- **接口 (Interface)**: 使用 `口` 定义方法契约。类只要实现了接口定义的所有方法，即可被视为该接口类型。
- **泛型 (Generics)**: 使用 `<T>` 定义泛型类或函数，支持类型参数化。支持使用 `承` 进行**泛型约束**，如 `<T 承 会飞的>`。
- **自定义类**: 支持使用类名作为类型注解。

支持的内置类型：`整`, `小数`, `字`, `逻`, `数组`, `字典`, `字节`, `结果`, `空`。

```xuantie
// 1. 泛型类
型 盒子<T> {
    设 值: T
    函 造(v: T) { 值 = v }
}
设 b: 盒子<整> = 造 盒子<整>(123)
//不支持的操作：目前不支持类型推导，必须显式书写。
设 b: 盒子<整> = 造 盒子(123)
//解释器拿不到具体的类型绑定信息，类定义中的 T 就会处于“未绑定”状态
//当它尝试校验传入的 123 （整型）是否符合 T 时，由于不知道 T 是什么，就会抛出类型不匹配的错误。

// 2. 泛型函数与约束
函 启动飞行<T 承 会飞的>(器物: T) {
    器物.飞行()
}
启动飞行<鸟>(造 鸟())

// 3. 接口定义
口 会飞的 {
    函 飞行()
}
```

### 运行时位置感知 (v0.9.6)
玄铁现在具备强大的错误追踪能力。当程序发生异常时（如除零、类型错误），会输出清晰的错误信息及**堆栈追踪 (Stack Trace)**，帮助你快速定位问题。

```xuantie
函 A() { B() }
函 B() { 1 / 0 }
A() 
// 报错将显示: 
// [第 X 行]: 除数不能为零
// 堆栈追踪:
//   于 B [行:Y]
//   于 A [行:Z]
```

### 索引与属性访问
支持通过 `[]` 访问数组、字典和字符串的元素，支持通过 `.长度` 获取容器大小。
```xuantie
设 文本 = "玄铁"
示(文本[0]) // 输出: 玄
示(文本.长度) // 输出: 2 (UTF-8 字节长度)

设 列表 = [1, 2, 3]
示(列表[1]) // 输出: 2
示(列表.长度) // 输出: 3
```

### 数组高级操作 (v0.9.3)
数组现在具备了工业级编程语言的成员方法：
- `.追加(元素1, ...)`：在末尾添加元素，支持多个。
- `.截取(起始, [结束])`：截取子数组（不改变原数组）。
- `.删(索引)`：删除指定位置元素。
- `.插(索引, 元素)`：在指定位置插入元素。
- `.找(元素)`：返回元素第一次出现的索引，未找到返回 -1。
- `.映射(回调(x, i))`：返回一个经过回调函数处理后的新数组。
- `.过滤(谓词(x, i))`：返回一个满足条件的元素组成的新数组。

```xuantie
设 列表 = [1, 2, 3]
列表.追加(4).删(0) // 链式操作：[2, 3, 4]
设 结果 = 列表.过滤(函(x) { 返 x > 2 }) // [3, 4]
```

### 逻辑运算
支持标准的逻辑运算，并具备**短路求值**特性。
- `且` (AND): 两个条件都为真时结果为真。
- `或` (OR): 其中一个条件为真时结果为真。
- `非` (NOT): 对逻辑值取反。
- `是` (IS): 用于判断类型或逻辑相等。
- `-` (负号): 对数值取反。
- `%` (取模): 获取除法的余数。

```xuantie
若 (1 > 0) 且 (2 > 1) { 示("真") }
设 结果 = 非 真 // 结果为假
若 1 等于 1 { 示("一等于一") }
若 1 是 整 { 示("1 是整数") }
```

## 3. 流程控制

### 条件判断 (若-抑-否)
玄铁采用极其简练的条件控制语法。
> `抑`古义为“或者”。在这里，它表示“但若条件为真则执行”。
```xuantie
若 10 > 5 {
    示("正确")
} 抑 10 == 5 {
    示("等于")
} 否 {
    示("错误")
}
```

### 循环与遍历
玄铁支持条件循环以及针对容器的快速遍历。

**当 (While) 循环**
```xuantie
设 i = 0
当 i < 3 {
    示(i)
    i = i + 1
}
```

**循 (Loop) 无限循环**
```xuantie
设 k = 0
循 {
    k = k + 1
    若 k > 10 { 断 }
    //如果k大于10那么断开（跳出）循环
}
```

**遍历 (For-In) 循环**
支持遍历数组、字典（键）以及字符串（字符）。
```xuantie
设 列表 = [10, 20, 30]
遍历 元素 于 列表 {
    示(元素)
}

设 映射 = {"名": "玄铁", "版": 0.6}
遍历 键 于 映射 {
    示(键 & ": " & 映射[键])
}
```

**循环控制**
- `断` (Break): 立即终止并跳出当前最内层循环。
- `续` (Continue): 跳过当前循环体剩余代码，直接进入下一次迭代。
```xuantie
遍历 i 于 [1, 2, 3, 4, 5] {
    若 i == 2 { 续 } // 跳过 2
    若 i == 4 { 断 } // 到 4 停止
    示(i)
}
```

设 加法 = 函(a, b) {

## 5. 进阶特性 (v0.3.0+)

### JSON 原生支持 (v0.7.0)
玄铁内置了极简的序列化与反序列化工具。
- **`化` (Serialize)**：将对象转化为 JSON 字符串。
- **`解` (Deserialize)**：将 JSON 字符串还原为玄铁对象（字典或数组）。

```xuantie
设 数据 = {"名": "玄铁", "值": 100}
设 文本 = 化(数据)  // 得到 JSON 字符串
设 对象 = 解(文本)  // 还原为字典
示(对象["名"])
```

### 字典 (Dict)
字典使用 `{键: 值}` 语法。
- **键**: 目前支持字符串作为键。
- **内置方法**:
    - `d.键()`: 返回所有键的数组。
    - `d.值()`: 返回所有值的数组。
    - `d.含(k)`: 判断字典是否包含键 `k`。
    - `d.删(k)`: 删除键 `k`。

```xuantie
设 库 = {"名": "玄铁", "岁": 1}
示(库.键()) // ["名", "岁"]
若 库.含("名") { 库.删("岁") }
```

### 类与对象
玄铁提供了轻量级的类与对象支持。
- **`型` (Type/Class)**：定义一个数据模型。
- **`造` (New/Construct)**：基于模型创建实例或定义构造函数。
- **`此` (This/Self)**：在类方法内部引用实例自身及属性。
- **`承` (Inherit)** (v0.8.0)：实现类之间的继承关系，子类可复用父类的属性和方法。
- **权限修饰符** (v0.7.2+)：
    - **`公` (Public)**：默认级别，外部可自由读写。
    - **`私` (Private)**：仅限本类内部访问，外部和子类均不可见。
    - **`护` (Protected)** (v0.8.0)：受保护级别，允许本类和子类访问，禁止外部直接访问。

#### 1. 基础定义与构造
```xuantie
型 坐标 {
    设 x = 0
    私 设 y = 0
    
    // 构造函数
    函 造(初始x, 初始y) {
        此.x = 初始x
        此.y = 初始y
    }
}
设 点 = 造 坐标(10, 20)
```

#### 2. 继承 (承) 与 保护 (护)
**`承`** 实现了代码复用，而 **`护`** 确保了敏感逻辑在“家族链”内部的安全流动。

```xuantie
型 账户 {
    公 设 户名 = "访客"
    护 设 余额 = 0  // 只有家族成员（子类）能动余额
    私 设 内部标记 = "SECRET" // 只有账户类自己能看
}

型 信用卡 承 账户 {  // 信用卡 继承了 账户
    函 存款(数额) {
        余额 = 余额 + 数额 // 合法：子类可以访问“护”属性
        示("存入成功，当前余额: " & 余额)
    }
}

设 我的卡 = 造 信用卡()
我的卡.存款(1000)
示(我的卡.户名)   // 合法：公有属性
// 示(我的卡.余额) // 报错：外部禁止访问受保护属性
```

### 网络与系统原语 (v0.9.0)
玄铁通过极简的单字原语，将复杂的网络通讯和系统交互降维到“管道读写”级别。

#### 1. 网络请求 (`求`)
一行代码获取互联网数据，原生返回 `结果` 容器。
```xuantie
求("https://api.github.com").接着(函(内容) {
    示("收到数据: " & 内容)
})
```

#### 2. Socket 与 HTTP 通讯 (`听` 与 `连`)
玄铁的 `听` 关键字具有多态性，能根据回调函数的参数数量自动切换 TCP 或 HTTP 模式。

- **TCP 模式**：回调函数接收 1 个参数 (`流`)。
```xuantie
听(9000, 函(管道) {
    设 数据 = 管道.读()
    管道.写("已收到")
    管道.关()
})
```

- **HTTP 模式**：回调函数接收 2 个参数 (`请`, `答`)。
```xuantie
听(8080, 函(请, 答) {
    示("收到请求: " & 请["方法"] & " " & 请["路径"])
    答.头("Content-Type", "text/plain")
    答.码(200)
    答.写("你好，玄铁 HTTP")
})
```

- **`连` (Connect)**：建立 TCP 客户端连接，返回 `结果(流)`。
- **`道` (Channel)** (v0.9.0)：创建并发通讯通道。
    - `设 通 = 道`：创建默认通道（缓冲大小 100）。
    - `通.发(值)`：向通道发送数据。
    - `通.收()`：从通道接收数据（阻塞）。
    - `通.关()`：关闭通道。
- **`流` (Stream)**：支持 `读`、`写`、`关` 操作。

#### 3. 系统执行 (`执`)
调用操作系统命令，并获取其输出。
```xuantie
执("git status").接着(函(输出) {
    示(输出)
})
```

#### 4. 通道通讯 (`道`)
在不同的异步任务（协程）之间安全地传递数据。
```xuantie
设 消息道 = 道
异步 { 消息道.送("秘密数据") }
设 收到 = 消息道.收()
```

### 字节处理 (字节) (v0.9.2)
用于处理二进制数据，支持与字符串互转。
- `字节.从字(s)`: 将字符串转为字节数组。
- `字节.到字(b)`: 将字节数组转为字符串。
- 属性 `.长度`: 获取字节数。
- 索引 `[i]`: 获取指定位置的字节值 (0-255)。

### 系统权能 (系统) (v0.9.2)
- `系统.参数`: 返回一个字符串数组，包含程序启动时的所有命令行参数。

### 外部函数接口 (外) (v0.9.1)
玄铁支持直接调用 C 语言编写的动态库（Windows `.dll` 或 Linux `.so`）。玄铁会自动识别以 `W` 结尾的 Windows 函数并使用 UTF-16 编码，彻底解决乱码问题。

```xuantie
外.加载("user32.dll").接着(函(库) {
    // 调用 MessageBoxW (Unicode版本)
    设 结果 = 库.MessageBoxW(0, "内容", "标题", 0)
    示("返回: " & 结果)
})
```

### 交叉编译
玄铁命令行支持一键构建跨平台程序。
> zao=造=编译 | pt=pingtai=平台 | jg=jiagou=架构
```bash
# 在 Windows 下编译 Linux 版本
xuantie zao 程序.xt --pt linux --jg amd64
```

### 结果处理 (Result)
引入函数式错误处理，避免嵌套的 `if err != nil`。
> 这种写法主要参照了 JavaScript (Promises) 、 Rust ( `Result<T, E>` )
```xuantie
设 获取数据 = 函() { 返 成功("数据") }

获取数据()
    .接着(函(内容) { 示("成功: " & 内容) })
    .否则(函(错误) { 示("失败: " & 错误) })
```
相比传统，如C语言的示例写法：
```
// C 语言风格：不断地检查返回状态
int err = get_data(&data);
if (err == SUCCESS) {
    int err2 = process_data(data, &result);
    if (err2 == SUCCESS) {
        save(result);
    } else {
        handle_error(err2);
    }
} else {
    handle_error(err);
}
```
这种写法会导致著名的“金字塔嵌套”或大量的 if (err != 0) goto cleanup 。
玄铁的解决方案是“管道化”：
 - `.接着`的本质： `获取数据()` 返回的不是一个简单的整数或指针，而是一个 结果 (Result) 容器。这个容器像一个包裹，里面可能装着“成功的数据”，也可能装着“失败的错误”。
 - 链式调用 ： `.接着` 是这个容器的一个方法。它的逻辑是：“如果你（容器）里面是成功的，就请执行我传给你的函数；如果你是失败的，请跳过我，把错误直接传给下一个人。”
 - 换行与点 ：这是一种 视觉对齐 。它让代码读起来像一个 流水线清单 ：
  1. 获取数据
  2. `.接着` 处理它
  3. `.接着` 转换它
  4. `.否则` 报错

### 异步与并行 (Async/Parallel)
原生支持并发，底层基于 Go 协程。
```xuantie
// 异步执行
设 任务 = 异步 { 耗时任务() }
设 结果 = 等待(任务)

// 并行执行多个块
设 结果集 = 并行 {
    { 任务1() },
    { 任务2() }
}
```

### 异常捕获 (Try/Catch)
捕获运行时错误（如除零、未定义变量）。
```xuantie
尝试 {
    10 / 0
} 捕捉 (错误) {
    示("捕获异常: " & 错误)
}
```

## 7. 内置标准库 (v0.3.1+)
玄铁开始构建其 IO 与网络能力。

### 文件操作 (`文件`) (v0.11.2)
```xuantie
// 读文件
文件.读("test.txt").接着(函(内容) {
    示(内容)
})

// 写文件
文件.写("test.txt", "你好玄铁").接着(函(成功) {
    示("写入成功")
})

// 打开文件句柄 (v0.11.2)

```

### 网络
- `网络.获取(URL)`: 返回 `结果`，包含网页正文内容。支持配合 `异步` 使用。

### 字符串处理
玄铁提供了强大的字符串处理内置函数。
- `字符串.长度(s)`: 获取字节长度。也可直接使用 `s.长度`。
- `字符串.包含(s, 子串)`: 判断是否包含子串。也可直接使用 `s.包含(子串)`。
- `字符串.子串(s, 起始, 长度)`: 截取子字符串。
- `字符串.替换(s, 旧, 新)`: 全量替换子串。
- `字符串.分割(s, 分隔符)`: 返回分割后的数组。

### 数学运算
- `数学.绝对值(n)`: 获取绝对值。
- `数学.正弦(n)`, `数学.余弦(n)`: 三角函数。
- `数学.平方根(n)`: 计算平方根。
- `数学.随机数()`: 返回 0.0-1.0 之间的浮点数。
- `数学.随机数(n)`: 返回 0-(n-1) 之间的整数。

### 时间处理 (时)
玄铁推荐使用极简的单字原语 `时` 进行时间操作。
- `时.现()`: 获取当前 Unix 时间戳（毫秒）。
- `时.睡(ms)`: 程序暂停指定的毫秒数。
- `时.格式化(ts, 格式)`: 将时间戳格式化为字符串。格式遵循 Go 的 layout (如 `2006-01-02 15:04:05`)。

### 模块化 (v0.3.2)
使用 `引` 关键字加载其他脚本，并获得其导出的变量。
- **路径解析**：相对路径（如 `./lib.xt`）是相对于**当前代码文件**所在的目录。这使得你可以轻松地将模块组织到不同的子目录中。
```xuantie
设 工具 = 引 "./utils.xt"
示(工具.加法(1, 2))
```

### 编译为二进制 (v0.3.7+)
玄铁支持将 `.xt` 脚本直接编译为原生的 `.exe` (Windows) 或二进制文件。
```bash
xuantie zao 你的程序.xt
```
- **优势**：执行速度大幅提升，不再依赖玄铁解释器，且源码已转换为机器码。
- **交叉编译** (v0.9.1)：支持使用 `--pt` (平台) 和 `--jg` (架构) 参数构建其他系统产物。
