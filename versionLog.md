次版本（Minor）：向后兼容功能/部分新功能/大更新

## v0.17.2 (2026-04-28)

- 补全了数学标准库：现在支持 `数学.绝对值`、`数学.正弦`、`数学.余弦`、`数学.平方根`、`数学.幂`、`数学.取整`、`数学.进一`、`数学.四舍五入` 等核心函数；
- 实现了变参数学函数支持：针对 `数学.最大值` 和 `数学.最小值` 实现了编译器级别的自动装箱，支持 `...n` 风格的无限参数调用；
- 注册了数学模块常量：现在可以直接通过 `数学.PI` 和 `数学.E` 访问高精度数学常量；
- 修复了自举词法分析器的标识符识别漏洞：支持以 `?` 结尾的标识符，顺利解决了 `文件.存在?` 被错误解析为脱壳运算导致的崩溃问题；
- 增强了 `?` 操作符的类型安全：在 IR 层增加了运行时类型校验，非 `结果` 对象调用该操作符时将触发安全报错而非段错误；
- 修正了 `xt_result_new` 的 ABI 调用约定：将成功标志参数由 `i1` 修正为 `i32`，提升了跨语言调用的传参稳定性；
- 补全了并发对象的析构逻辑：在运行时中完善了 `通道` (Channel)、`任务` (Task) 和 `字节流` (Bytes) 的内存释放逻辑，消除了长时间运行时的内存泄漏隐患；

## v0.17.1 (2026-04-28)

- 新增了位运算操作符支持：现在支持 `位与`、`位或`、`异或`、`左移`、`右移` 和前缀 `取反` 关键字；
- 实现了自举编译器的位运算 IR 生成：针对标记指针（Tagged Pointer）进行了优化，位运算会自动处理脱壳与装壳逻辑，确保类型安全；
- 同步更新了 Go 种子编译器：现在种子编译器也能识别并编译位运算操作符，确保了自举链路的一致性；
- 优化了位运算的执行效率：在 LLVM 后端为位运算生成了高效的原生指令，并支持标量优化（Raw i64）下的零开销计算；

## v0.17.0 (2026-04-27)——自举驱动加固+MinGW深度集成+全面性能突破

> 注：所有更新均仅在Windows平台测试过，Linux和Darwin平台未进行任何测试。

- **全面切换至 MinGW 工具链**：种子编译器和自举编译器现在默认生成 MinGW 格式的对象文件，并使用 `gcc` 进行链接，彻底摆脱了对 MSVC 运行时库的依赖；
- **全面硬化了运行时 C 代码**：将所有潜在风险的 `sprintf` 迁移至 `snprintf`，确保在处理超长 URL、错误消息或文件路径时的缓冲区安全性；
- 自举编译器成功驱动化，支持直接编译玄铁程序；
- 新增了原生 FFI 接口 `xt_get_temp_path()`：直接封装 Win32 `GetTempPathA` 极速获取系统临时目录，移除了驱动层之前低效的 `cmd /c echo %TEMP%` 子进程调用与字符串清洗逻辑；
- 新增了单引号和多行字符串支持，现在用 `'''` 就能直接写带换行的长文本，插值 `#{ }` 也是全通的；
- 新增了范围表达式（比如 `1..5`），配合 `遍历` 循环特别好用，底层会自动帮你造好数组；
- 新增了解构赋值语法，不管是数组提取 `[首, ...余]` 还是字典提取 `{名, 别}` 都能一气呵成；
- 新增了早返回操作符（`?`），碰到 `结果` 报错直接往上甩，省去了大段的逻辑判断；
- 新增了自举编译器的输出控制参数 `-sc` / `--输出`，现在能自由指定生成的二进制文件名了；
- 新增了自举编译器的详细信息输出参数 `-xx` / `--xinxi` / `--信息`，默认情况下保持简洁输出，仅在需要调试时显示底层指令；
- 优化了 Windows 指令执行 (Driver Engine)：弃用了直接 `_wpopen` 调用，改为通过唯一性命名（PID + 毫秒时间戳）的临时 `.bat` 文件执行，彻底解决了 Windows 下的引号剥离（Quote Stripping）与并发文件碰撞隐患；
- **优化了 argv 获取方式 并 顺利修复了获取中文 argv参数 乱码的问题**：之前走main的argv，拿到的都是ANSI（GBK）字节，要么乱码要么得做不可靠的GBK转UTF-8。现在LLVM-IR里的main会直接`call xt_init_args(argc, argv)`，`xt_init_args`调用 `GetCommandLineW()` + `CommandLineToArgvW()` 直接拿UTF-16无损转换为UTF-8，中文文件名和路径不会再乱码；
- 优化了字符串核心操作的编译方式，把 `字节数`、`取字节`、`字符` 这些常用方法直接硬编码进后端，避开了自举时容易踩坑的动态查找；
- 优化了符号生成安全性，增加了 `符()` 和 `标()` 包装函数，彻底封锁了“空”值被误当成字符串塞进 IR 的漏洞；
- 优化了跨模块的类型解析，现在支持继承别的模块里的类（`型 A 承 别名.B`），实例化也支持带前缀的写法了；
- 优化了 CLI 参数解析逻辑，支持单横杠短参数（如 `-sj`）与双横杠长参数（如 `--shijian`）；
- 优化了成员访问报错，现在跨模块访问不存在的成员时会报出精准的错误信息，并立即终止编译；
- 更新了 LLVM Target Triple 为 `x86_64-w64-windows-gnu`，Data Layout 调整为 MinGW 兼容格式；
- 优化了 FFI 调用稳定性：引入了 `xt_ffi_printf` 安全包装器，解决了 MinGW 环境下跨 ABI 传递字符串对象导致的乱码和崩溃问题；
- 增强了运行时字符编码兼容性：支持现代 Windows 的 `.UTF8` 区域设置，确保中文字符串在 FFI 场景下的精准显示；
- 增强了 `执()` 指令的安全性：为指令路径增加了 1024 字节的长度校验与 `_snwprintf` 安全格式化，若路径超限将通过 `结果` 容器返回明确的错误信息，而非导致程序崩溃；
- 增强了前端错误拦截机制：
  - 词法期严判：增强了词法分析器对特殊字符的过滤，现在遇到 `^`、`@`、`$` 等未定义操作符时会立即报错并终止，防止非法字符作为标识符流向后端；
  - 编译期语义熔断：在自举编译器的语义分析与 IR 生成阶段，引入了严格的“错误熔断”机制。凡是遇到“未知标识符”、“未定义类型”或“越权成员访问”，编译器将立即调用 `终 1` 强制退出，彻底解决了之前错误 IR 泄露导致 LLVM 报错难懂的问题；
- 优化了链接时的文件锁处理：在自举驱动中增加了输出文件预删除逻辑，解决了 Windows 下因输出二进制文件被占用导致的链接挂起与 5~10 秒超时报错问题；
- **修复了 Arena 销毁时的挂起与堆损坏问题**：
  - **结构体安全化**：为 `XTArena` 增加了 `XTObject` 头部，并设为“长生不老”状态，彻底解决了 ARC 引擎误将 Arena 句柄当作普通对象进行 `release` 导致的内存污染；
  - **扩容算法重构**：优化了 Arena 的自动扩容逻辑，采用“头部锚定”策略，确保用户持有的原始句柄始终指向链表头，使得 `xt_arena_destroy` 能够百分之百回收所有扩容块；
- **实现了 Windows 全链路 UTF-8 路径支持**：
  - 在运行时中增加了 `xt_utf8_to_utf16` 与 `xt_utf16_to_utf8` 转换原语；
  - 将 `文件.读`、`文件.写`、`文件.存在?`、`xt_get_temp_path` 以及 `执()` 指令全面迁移至 Windows 原生宽字符 API（如 `_wfopen`, `_wstat64`, `GetTempPathW`）；
  - 解决了 Windows 环境下因 ANSI 编码强制转换导致的中文字符路径乱码及文件操作失败问题；
- 修复了命令行参数的引用计数泄漏：在 `xt_init_args` 初始化全局参数数组后，增加了缺失的 `xt_release` 调用，确保了进程启动阶段内存管理的严格平衡；
- 优化了自举编译器的相对路径解析：现在子模块的 `引` 路径将基于当前源文件所在目录进行解析，而非基于编译器的工作目录，显著提升了大型多层目录项目的组织灵活性；
- 优化了基准测试文案与反馈质量：
  - 更新了性能测试文案，将原本的“循环性能测试”明确标注为“LLVM 闭式优化演示”，以准确传达编译器代数简化的优化效果；
  - 补全了运行时所需的 `<sys/stat.h>` 与 `<sys/types.h>` 系统头文件，确保了 MinGW 环境下对 `_stat64` 等结构的编译兼容性；
- 补全了运行时里关于字符串取字节、数组切片和范围生成的底层函数；
- 修复了跨模块调用方法莫名其妙返回 `空` 的 Bug，原因是引入模块时漏掉了类定义的合并，现在已经修好了；
- 修复了生成的 IR 里中文字符串长度不对导致报错的问题，所有字符串常量现在都强行走 `.字节数` 计算，出来的长度百分之百精确；
- 修复了拼接符号（`&`）在自举环境下碰到 `空` 值会产生非法指令的隐蔽 Bug，并在拼接和插值时增加了明确的编译期警告；
- 修复了库引用失败时编译器继续运行产生级联报错的问题，现在会立即终止编译；
- 修复了 `XTResult` 结构体的 ABI 对齐问题：在 C 层增加了显式 `padding` 字段以匹配 x64 对齐要求，并同步修正了 LLVM IR 中的 GEP 索引，彻底解决了寻址错位导致的返回值丢失问题；
- **新增高精度计时原语 `时.微秒()`**：
  - 在运行时层实现了基于 `QueryPerformanceCounter` (Windows) 和 `clock_gettime` (Linux) 的亚微秒级计时接口；
  - 允许开发者通过微秒级数据更精准地评估代码段的性能，彻底告别“0 毫秒”误差；
- **[性能突破] 引入了局部整数脱壳优化 (Raw i64 Optimization)**：
  - 对局部整数变量（如循环计数器）引入了基础类型推断，直接使用原生 `i64` 存储而非标记指针（Tagged Pointer）；
  - **零成本算术运算**：对于纯整数运算，跳过了繁琐的“脱壳-运算-装壳”流程，直接生成原生 LLVM 加减乘除指令；
  - **ARC 消除 (Scalar ARC Elimination)**：
    - 对识别为 `raw_i64`、`i1`（布尔）等标量类型的局部变量，彻底移除了读取时的 `xt_retain` 和赋值时的 `xt_release` 调用；
    - **运行时加速**：在 `xt_retain/xt_release` C 函数内部增加了针对标记指针的快速返回路径，极大减少了无效的原子操作开销；
  - **延迟装壳机制**：仅在整数“逃逸”到通用容器（数组/字典）或跨函数调用时才进行装壳转换，大幅提升了密集型循环的执行效率（预计提升 2-5 倍）；
  - **LLVM 闭式求解优化突破**：由于 `raw_i64` 与 **Scalar ARC 消除** 彻底净化了算术热点路径的 IR 表现（无装拆箱、无冗余原子操作调用），LLVM 优化器现在能将复杂的 10⁸ 次方级线性递推循环识别并压缩为闭式解。在 1 亿次迭代测试中，执行耗时仅为 **11 微秒**（约 27 次迭代开销），标志着玄铁的整数代码质量已达到能让 LLVM 进行深度代数简化的原生级别；
  - **mem2reg 友好型 IR 生成**：通过消除标量类型的 `retain/release` 调用，确保局部变量的 `alloca` 地址不被外部函数引用，从而完美触发 LLVM 的 `mem2reg` 优化 pass，将局部变量全量提升至 SSA 寄存器，实现真正的内存 zero-copy 计算；
  - **算术强度削减 (Strength Reduction)**：
    - **取负优化**：针对标记指针（Tagged Pointer）的取负操作，由原本的 4 条指令（脱壳 -> 取负 -> 移位 -> 装壳）削减为 2 条原生指令：`sub i64 2, %val` 与 `or i64 %res, 1`。这一改进利用了标记指针的代数特性，大幅降低了单目运算的指令密度；
    - **标量短路路径**：前端 AST 现在能识别已脱壳的 `raw_i64` 环境，直接生成最简化的原生算术 IR，不再盲目执行装拆箱操作；
