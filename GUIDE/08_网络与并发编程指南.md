> **版本**：v0.2.0 | **状态**：异步/并行/网络栈已落地，异步闭包捕获为已知局限

## 8.1 并发模型

玄铁采用 **M:N 线程池** 模型——M 个 OS 线程执行 N 个用户任务，类似 Go 的 goroutine。

### 8.1.1 异步任务

`异步 { ... }` 将代码块提交到线程池独立执行，立即返回任务句柄。`等待(任务)` 阻塞等待结果。

```text
设 task = 异步 {
    示("任务执行中...")
    时.睡(1000)  // 模拟耗时操作
    示("任务完成")
}
示("主线程继续执行...")
等待(task)
示("任务已结束")
```

**执行流程**：
1. `异步` 块被编译为独立 LLVM 函数
2. `xt_async_spawn` 将函数提交到线程池
3. 主线程继续执行
4. `等待` 调用 `xt_async_wait` 阻塞等待任务结束

### 8.1.2 并行执行

`并行 { {块1}, {块2}, ... }` 并发执行多个代码块，全部完成后返回结果数组。

```text
设 results = 并行 {
    { 时.睡(500); 返 "A" }
    { 时.睡(300); 返 "B" }
    { 返 "C" }
}
// results = ["A", "B", "C"]
```

### 8.1.3 已知局限

- **异步闭包捕获**：顶层作用域的异步块可以捕获外部变量。嵌套在回调函数/匿名函数内的异步块暂不支持捕获外部变量（语义分析 pass 需同步修改）。

```text
// 顶层作用域——正常
设 name = "test"
异步 { 示(name) }  // ✓ 可以捕获 name

// 嵌套回调——暂不支持
听(8000, 函(流) {
    设 user = "guest"
    异步 { 示(user) }  // ✗ 暂不支持
})
```

## 8.2 通道

`道(容量)` 创建带缓冲的并发通道，支持 `发`（send）和 `收`（receive）。

### 8.2.1 基本用法

```text
设 ch = 道(4)      // 容量为 4 的通道
ch.发("hello")     // 发送
ch.发("world")
设 msg = ch.收()   // 接收
示(msg)
```

### 8.2.2 异步生产者-消费者

```text
设 ch = 道(3)

设 producer = 异步 {
    ch.发(1)
    ch.发(2)
    ch.发(3)
}

设 consumer = 异步 {
    设 sum = 0
    设 i = 0
    当 i < 3 {
        sum = sum + ch.收()
        i = i + 1
    }
    示("Sum: " & sum.转字())
}

等待(producer)
等待(consumer)
```

### 8.2.3 线程安全

通道操作（`发`/`收`）已加互斥锁保护，多线程并发安全。

## 8.3 HTTP 请求

`求(url)` 发起 HTTP GET 请求，返回 `结果` 对象。

```text
设 resp = 求("http://api.example.com/data")
若 resp.成功 {
    示("响应: " & resp.值)
} 否 {
    示("请求失败: " & resp.错误)
}
```

**当前支持**：HTTP/1.1 GET，自动解析 host:port/path，返回响应体文本。  
**暂不支持**：HTTPS、POST/PUT 等方法、自定义 Header。

## 8.4 TCP 网络

### 8.4.1 服务端（监听）

`听(端口, 回调函数)` 在指定端口启动 TCP 监听。每个新连接触发回调，传入 `流`（Socket 对象）。

```text
听(8000, 函(流) {
    示("新连接！")
    流.写("欢迎！\n")
    
    循 {
        设 data = 流.读()
        若 data == 空 { 断 }  // 连接关闭
        示("收到: " & data)
        流.写("回显: " & data)
    }
})

// 主线程继续运行
输("按回车停止...")
```

### 8.4.2 客户端（连接）

`连(地址)` 连接到 TCP 服务端，返回 `结果<XTSocket>`。

```text
设 conn = 连("127.0.0.1:8000")
若 conn.成功 {
    设 流 = conn.值
    流.发("hello")
    示("服务端回复: " & 流.读())
    流.关()
} 否 {
    示("连接失败: " & conn.错误)
}
```

### 8.4.3 Socket 方法

| 方法 | 说明 | 示例 |
|------|------|------|
| `.读()` | 读取数据，返回字符串或空 | `设 msg = 流.读()` |
| `.写(内容)` | 写入字符串 | `流.写("hello")` |
| `.发(内容)` | 同 `.写()` | `流.发("hello")` |
| `.关()` | 关闭连接 | `流.关()` |

### 8.4.4 Socket 生命周期

Socket 对象通过 `xt_release` 纳入 ARC 管理。引用计数归零时自动关闭底层连接。一般情况下不需要手动调 `.关()`——当 socket 变量离开作用域时自动清理。

```text
{
    设 流 = 连("example.com:80").值
    流.写("GET / HTTP/1.0\r\n\r\n")
    示(流.读())
    // 流 离开作用域，自动关闭
}
```

## 8.5 即时通讯 (IM) 完整示例

项目 `examples/im/` 包含完整的服务端+客户端示例。

### 服务端 (server.xt)

```text
常 端口 = 8000

函 处理连接(流, 初始名字) {
    设 名字 = 初始名字
    流.写("系统: 欢迎进入聊天室！")
    循 {
        设 数据 = 流.读()
        匹配 数据 {
            是 空 -> { 示("连接断开"); 断 }
            _ -> { 示("[" & 名字 & "]: " & 数据) }
        }
    }
}

听(端口, 函(流) {
    示("新连接")
    处理连接(流, "匿名用户")
})

示("服务端已就绪，监听端口 " & 端口.转字())
循 { 设 cmd = 输("管理>"); 若 cmd == "退出" { 断 } }
```

### 客户端 (client.xt)

```text
设 目标 = "127.0.0.1:8000"
设 名字 = 输("用户名: ")
若 名字 == "" { 名字 = "匿名用户" }

设 连接结果 = 连(目标)
若 连接结果.成功 {
    设 流 = 连接结果.值
    流.发("[用户名]:" & 名字)
    循 {
        设 msg = 输("我: ")
        若 msg == "退出" { 流.关(); 断 }
        流.发(msg)
    }
} 否 {
    示("连接失败")
}
```

编译运行：

```bash
# 终端1
.\玄铁_go.exe tie examples\im\server.xt
.\examples\im\server.exe

# 终端2
.\玄铁_go.exe tie examples\im\client.xt
.\examples\im\client.exe
```

## 8.6 线程安全速查

| 组件 | 线程安全性 | 说明 |
|------|-----------|------|
| 通道 (`道`) | **安全** ✓ | `xt_chan_mutex_t` 全程持锁 |
| 异步 (`异步`) | **安全** ✓ | 每任务独立栈+独立 env（若捕获） |
| 并行 (`并行`) | **安全** ✓ | cloneEnv 隔离 |
| Socket I/O | **安全** ✓ | 每个 socket 独立，无共享状态 |
| ARC (`xt_retain/release`) | **安全** ✓ | atomic_fetch_add/sub |
| 全局变量 | **不安全** ✗ | 用户代码需自行加锁保护可变全局变量 |
| 弱引用 | **安全** ✓ | `g_weak_mutex` 保护链表操作 |

---

## 8.7 性能建议

1. **异步优于并行**：简单任务用 `异步`，比 `并行` 开销更小
2. **通道容量**：根据生产者-消费者速率差选择合适的缓冲大小
3. **Socket 复用**：长时间连接保持 socket 存活，避免频繁连接/断开
4. **避免全局可变状态**：在异步/并行块中优先使用局部变量和通道通信
