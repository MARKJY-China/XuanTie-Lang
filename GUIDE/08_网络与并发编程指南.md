> **版本**：v0.5.0 | **状态**：异步/并行/通道/状态机/调度器/选已落地

## 8.1 并发模型

玄铁采用 **双层架构**：

| 层级 | 机制 | 说明 |
|------|------|------|
| OS 线程层 | 工作窃取线程池 | 执行 CPU 密集型任务，线程数 = CPU 核数 |
| 用户态调度层 | 无栈状态机 + 调度器 | 管理 I/O 等待任务，就绪队列 + 定时器堆 |

`异步 { ... }` 如果块内包含 `等待`/`收`/`发`/`选`，则编译为**无栈状态机**，由用户态调度器管理；否则编译为独立函数提交到线程池。

### 8.1.1 异步任务

`异步 { ... }` 将代码块作为并发任务执行，立即返回任务句柄。

```text
设 task = 异步 {
    示("任务执行中...")
    时.睡(1000)
    示("任务完成")
}
示("主线程继续执行...")
等待(task)
示("任务已结束")
```

**执行流程**：
1. 编译器检测异步块内是否有 `等待`/`收`/`发`/`选`
2. 有 → 编译为状态机（poll 函数 + 状态结构体），entry wrapper 调用 `xt_scheduler_spawn` 提交到调度器
3. 无 → 编译为独立函数，调用 `xt_async_spawn` 提交到线程池
4. 主线程执行到 `xt_scheduler_run()` 时启动调度器事件循环
5. `等待` 调用 `xt_async_wait` 阻塞等待任务结束（或从调度器挂起）

### 8.1.2 并行执行

`并行 { {块1}, {块2}, ... }` 并发执行多个代码块。

```text
设 results = 并行 {
    { 时.睡(500); 返 "A" }
    { 时.睡(300); 返 "B" }
    { 返 "C" }
}
// results = ["A", "B", "C"]
```

### 8.1.3 已知局限

- **异步闭包捕获**：顶层作用域的异步块可以捕获外部变量。嵌套在回调函数/匿名函数内的异步块暂不支持捕获外部变量。
- **循环变量**：异步块内使用 `遍历 i 于 arr` 的循环变量 `i` 会导致全局符号冲突，应避免或将循环体封装为独立函数。

## 8.2 通道

`道(容量)` 创建带缓冲的并发通道。v0.3 起支持**阻塞收发**（条件变量）。

### 8.2.1 基本用法

```text
设 ch = 道(4)      // 容量为 4 的通道
ch.发("hello")     // 非阻塞发送（满则丢弃）; 状态机中为阻塞发送
ch.发("world")
设 msg = ch.收()   // 非阻塞接收（空则返回空）; 状态机中为阻塞接收
示(msg)
```

### 8.2.2 阻塞收发（状态机内）

在 `异步` 块内使用 `.发()` / `.收()` 时，编译器自动使用阻塞版本：

```text
设 ch = 道(3)

设 consumer = 异步 {
    设 sum = 0
    设 i = 0
    当 i < 3 {
        sum = sum + ch.收()  // 阻塞等待，不占 OS 线程
        i = i + 1
    }
    示("Sum: " & sum.转字())
}

设 producer = 异步 {
    ch.发(1)
    ch.发(2)
    ch.发(3)
}

等待(producer)
等待(consumer)
```

### 8.2.3 多通道选择（选）

`选(通道数组, 超时毫秒)` 等待多个通道中第一个就绪的，返回通道索引。超时返回 -1。

```text
设 fast = 道(1)
设 slow = 道(1)

设 sel = 异步 {
    设 idx = 选([fast, slow], 5000)
    若 idx == 0 { 示("fast 就绪") }
    抑 idx == 1 { 示("slow 就绪") }
    否 { 示("超时") }
}
```

### 8.2.4 线程安全

通道操作（`发`/`收`/`选`）已加互斥锁+条件变量保护，多线程并发安全。

## 8.3 HTTP 请求

`求(url)` 发起 HTTP GET 请求，返回 `结果` 对象。

```text
设 resp = 求("http://api.example.com/data")
若 resp.成功 {
    示("响应: " & resp.值)
} 否 {
    示("请求失败: " & resp.错误)
}
```

**当前支持**：HTTP/1.1 GET，自动解析 host:port/path，返回响应体文本。  
**暂不支持**：HTTPS、POST/PUT 等方法、自定义 Header。

## 8.4 TCP 网络

### 8.4.1 服务端（监听）

`听(端口, 回调函数)` 在指定端口启动 TCP 监听。

```text
听(8000, 函(流) {
    示("新连接！")
    流.写("欢迎！\n")
    循 {
        设 data = 流.读()
        若 data == 空 { 断 }
        示("收到: " & data)
        流.写("回显: " & data)
    }
})
输("按回车停止...")
```

### 8.4.2 客户端（连接）

`连(地址)` 连接到 TCP 服务端，返回 `结果`。

```text
设 conn = 连("127.0.0.1:8000")
若 conn.成功 {
    设 流 = conn.值
    流.发("hello")
    示("服务端回复: " & 流.读())
    流.关()
} 否 {
    示("连接失败: " & conn.错误)
}
```

### 8.4.3 Socket 方法

| 方法 | 说明 |
|------|------|
| `.读()` | 读取数据，返回字符串或空 |
| `.写(内容)` | 写入字符串 |
| `.发(内容)` | 同 `.写()` |
| `.关()` | 关闭连接 |

## 8.5 即时通讯 (IM) 完整示例

项目 `examples/im/` 包含完整的服务端+客户端示例。

```text
// 服务端
听(8000, 函(流) {
    示("新连接")
    流.写("系统: 欢迎进入聊天室！")
    循 {
        设 数据 = 流.读()
        若 数据 == 空 { 断 }
        示("收到: " & 数据)
    }
})
示("服务端已就绪")
输("按回车停止...")
```

## 8.6 线程安全速查

| 组件 | 线程安全性 | 说明 |
|------|-----------|------|
| 通道 (`道`) | **安全** ✓ | `xt_chan_mutex_t` + 条件变量全程持锁 |
| 通道（阻塞收发） | **安全** ✓ | 条件变量等待，不占 OS 线程 |
| 选 (`选`) | **安全** ✓ | 轮询+条件变量，超时安全 |
| 异步 (`异步`) | **安全** ✓ | 状态机 poll 函数由调度器单线程调用 |
| 并行 (`并行`) | **安全** ✓ | cloneEnv 隔离 |
| Socket I/O | **安全** ✓ | 每个 socket 独立，无共享状态 |
| ARC | **安全** ✓ | atomic_fetch_add/sub |
| 调度器 | **注意** ⚠ | 线程池回调唤醒 fiber，无锁竞态（当前单线程未触发） |
| 全局变量 | **不安全** ✗ | 用户代码需自行加锁 |
| 弱引用 | **安全** ✓ | `g_weak_mutex` 保护 |
