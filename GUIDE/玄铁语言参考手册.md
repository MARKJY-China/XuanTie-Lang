# 玄铁语言参考手册 (XuanTie Language Reference Manual)

> **导读**：本手册主要面向语言使用者（层次 1：语法层），旨在全面盘点玄铁语言的关键字、类型系统、控制流、面向对象机制、模块系统、包管理器（铁铺）、渲染库（2D 桌面开发）以及标准库功能。玄铁采用全中文关键字设计，力求提供贴近自然语言思维的编程体验。

## 1. 词法与基础语法

### 1.1 核心关键字 (Keywords)

玄铁的关键字体系涵盖了从变量声明到并发控制的全方位支持：

*   **声明与作用域**：
    *   `设`：声明变量 (var)
    *   `常`：声明常量 (const)
    *   `函`：声明函数 (func)
    *   `返`：函数返回 (return)
    *   `外`：声明外部函数/C-FFI (external)
    *   `型`：定义类/类型 (type)
    *   `口`：定义接口 (interface)
    *   `造`：实例化/构造函数 (new)
    *   `承`：继承父类 (inherit)
    *   `覆`：重写父类方法 (override)
    *   `弱`：弱引用修饰符
    *   `此`：当前实例引用 (this)
    *   `引`：引入外部模块 (import)
    *   `予`：为模块指定别名 (alias)
*   **并发与异步**：
    *   `异步`：声明异步任务 (async)
    *   `等待`：等待异步结果 (await)
    *   `并行`：并行执行 (parallel) *（关键字已定义，编译器支持待完善）*
    *   `道`：通道 (channel)
*   **异常与匹配**：
    *   `尝试` / `捕捉`：异常捕获 (try/catch)
    *   `匹配`：模式匹配 (match)
    *   `终`：终止程序/抛出异常 (terminate)
*   **系统级内建原语**：
    *   `示`：打印输出 (print)
    *   `化`：JSON 序列化
    *   `解`：JSON 反序列化
    *   `执`：执行系统命令 (exec)
    *   `输`：读取标准输入 (input)
    *   `求`：发起 HTTP 请求
    *   `听`：网络监听
    *   `连`：建立网络连接

### 1.2 运算符 (Operators)

*   **算术**：`+`, `-`, `*`, `/`, `%`
*   **比较**：`==`, `!=`, `<`, `>`, `<=`, `>=`
*   **逻辑**：`且` (AND), `或` (OR), `非` (NOT), `是` (IS)
*   **位运算**：`位与`, `位或`, `异或`, `左移`, `右移`, `取反`
*   **其他**：
    *   赋值：`=`
    *   成员访问：`.`
    *   箭头：`->`（匹配语句中模式与分支之间的分隔符：`模式 -> 块`）
    *   **错误冒泡后缀操作符**：`?`（跟在返回 `结果` 类型的表达式之后，用于快速解包或向上传递错误，如 `设 值 = 可能出错的操作()?`）

### 1.3 字面量与标识符特性 (Literals & Identifiers)

*   **布尔值**：`真` (true), `假` (false)
*   **空值**：`空` (null)
*   **数字**：支持标准的十进制整数与浮点数（小数）
*   **字符串**：支持双引号 `""` 或单引号 `''`
*   **问号标识符**：标识符支持以 `?` 结尾，常用于返回布尔值的函数或变量命名（例如 `文件.存在?`、`队列.空?`），这让代码读起来更像自然语言的提问。

## 2. 类型系统现状

玄铁具备完备的静态类型关键字。目前在编译期与运行时实际发挥作用的核心类型如下：

*   **基本数据类型**：
    *   `整` (Int/i64)：整数类型，广泛用于算术运算、索引等。
    *   `小数` (Float/double)：浮点数类型。
    *   `字` (String/%XTString)：字符串对象引用。
    *   `判` (Bool/i1)：布尔类型，用于控制流和条件判断。
*   **复合/数据结构类型**：
    *   `数组` (Array/%XTArray)：动态数组引用。
    *   `字典` (Dict/%XTDict)：哈希表/字典引用。
*   **高级抽象类型**：
    *   `结果` (Result)：用于包装操作成功/失败的状态类型（详见 [01_结果类型与错误处理.md](./01_结果类型与错误处理.md)）。
    *   `任务` (Task)：异步执行的任务句柄。
    *   `字节` (Bytes)：用于底层内存、文件或网络 I/O 操作的原始字节流。

### 2.1 泛型 (Generics)

玄铁支持在 `型` 和 `函` 中定义泛型参数，使用 `<T>` 语法。泛型参数可以是任意标识符，实例化时传入具体类型（如 `<整>`、`<字>`）。当前编译器采用类型擦除策略，泛型主要提供语法层面的类型参数化能力。

*   **泛型类**：`型 容器<T> { 设 内容: T ... }` — 定义可复用容器类
*   **泛型函数**：`函 交换<T>(a: T, b: T) { 返 [b, a] }` — 定义类型参数化函数
*   **实例化**：`造 容器<整>(123)` / `交换<整>(10, 20)` — 传入具体类型参数
*   **可空修饰**：`?` 紧跟在类型后，修饰紧邻类型——`节点?`（节点可空）、`数组<节点?>`（元素可空）、`数组<节点>?`（数组可空）

> 详细语法说明、教程与示例代码请参阅：[03_泛型编程指南.md](./03_泛型编程指南.md)

## 3. 控制流体系

玄铁的控制流设计贴合中文语境，主要包含以下几种结构：

### 3.1 条件分支
使用 `若`、`抑` (else if)、`否` (else) 进行逻辑判断：
```text
若 条件A {
    // ...
} 抑 条件B {
    // ...
} 否 {
    // ...
}
```

### 3.2 循环结构
*   **条件循环** (`当`)：
    ```text
    当 条件 { ... }
    ```
*   **无限循环** (`循`)：
    ```text
    循 { 
        若 退出条件 { 断 } 
    }
    ```
*   **迭代循环** (`遍历`)：搭配 `于` (in) 关键字使用，常用于遍历数组或字典：
    ```text
    遍历 元素 于 集合 { ... }
    ```
*   **循环控制**：`断` (break) 退出循环；`续` (continue) 跳过本次循环。

## 4. 面向对象与类系统

玄铁支持基于类的面向对象编程，包含封装、继承与多态机制：

*   **类的定义与构造**：
    *   使用 `型` 声明一个类。
    *   使用 `造` 关键字定义构造函数（或用于实例化对象）。
*   **实例与成员访问**：
    *   使用 `此` 关键字来访问当前实例的方法和字段（相当于 `this`）。
*   **可见性控制**：
    *   支持 `公` (public)、`私` (private)、`护` (protected) 修饰符，用于控制方法或字段的作用域。
*   **继承体系**：
    *   `承`：用于继承父类 (inherit)。
    *   `覆`：用于重写 (override) 父类的方法。
*   **弱引用**：
    *   使用 `弱 设` 声明弱引用变量或类成员字段，打破 ARC 循环引用。
    *   弱引用不持有对象所有权——当所有强引用释放后，弱引用自动置为 `空`。
    *   搭配可空类型 `?` 使用：`弱 设 父: 父节点?`。

> 详细语法说明、ARC 行为与示例代码请参阅：[04_弱引用编程指南.md](./04_弱引用编程指南.md)

## 5. 模块系统

玄铁的模块系统采用直接的引入和命名空间机制：

*   **引** (Import)：引入外部模块。路径相对于当前源文件所在目录。
    ```text
    引 "./库/数学.xt"
    引 "./库/文件.xt" 予 文件
    ```
*   **予** (Alias)：为引入的模块指定别名，作为访问其成员的前缀。
*   **命名空间隔离**：不同模块的同名符号通过模块前缀区分，不会冲突。
*   **编译机制**：预扫描阶段递归收集所有引入模块的符号，合并编译为单一可执行文件。循环引用被检测并报错。
*   **路径规范**：玄铁强制路径分隔符规范——**`/` 代表相对路径，`\` 代表绝对路径**（Windows 原生）。编译器自动将混用斜杠规范化：相对路径中 `\` → `/`，绝对路径中 `/` → `\`。

> 详细语法、路径解析、完整示例请参阅：[05_模块系统.md](./05_模块系统.md)

### 5.1 铁铺包管理器 (TiePM)

铁铺是玄铁语言的官方包管理器，提供铁器的搜索、安装与发布功能：

*   **安装铁器**：`铁铺 安装 <包名>` — 从远程索引下载并验证安装
*   **搜索铁器**：`铁铺 搜索 <关键词>` — 按名称和描述搜索可用包
*   **发布铁器**：`铁铺 发布` — 打包当前项目并登记到官方索引
*   **包清单**：项目根目录的 `tiepm.toml` 文件描述包的元信息（名称、版本、仓库、哈希）
*   **安全保障**：所有下载文件经过 SHA256 哈希验证，确保完整性

> 完整使用指南、命令参考、发布工作流请参阅：[07_铁铺使用指南.md](./07_铁铺使用指南.md)
>
> 架构设计与 API 规范请参阅：[铁铺设计笔记.md](./铁铺设计笔记.md)

## 6. 标准库与 Runtime API

玄铁在底层 Runtime (`xt_runtime.c`) 提供了丰富的内置函数和标准库对象方法，这些 API 可以直接在语言层被调用：

### 6.1 全局内置原语
*   **基础交互**：`示(内容)` (等同于 Print)
*   **数据转换**：`化(对象)` (JSON序列化)，`解(字符串)` (JSON反序列化)
*   **系统与网络**：
    *   `执(命令)`：执行系统命令 (Execute)
    *   `输(提示)`：读取标准输入 (Input)
    *   `听(端口, 处理函)`：网络监听 (Listen)
    *   `求(地址)`：发起 HTTP 请求 (Request)
    *   `连(地址)`：建立网络连接 (Connect)

### 6.2 标准库命名空间
*   **文件 (`文件.`)**：
    *   `文件.读(路径)`
    *   `文件.写(路径, 内容)`
    *   `文件.存在?(路径)`
*   **数学 (`数学.`)**：
    *   `数学.随机()`: 获取随机数
*   **时间 (`时.`)**：
    *   `时.现()`: 获取当前时间戳
    *   `时.毫秒()`, `时.微秒()`: 获取高精度时间
    *   `时.睡(时长)`: 线程休眠 (Sleep)

### 6.3 数组

数组支持动态扩容、索引访问和丰富内置方法。创建：`设 列表 = [1, 2, 3]`；索引：`列表[0]`；长度：`列表.长度`。

**内置方法**：`追加`(append)、`删`(remove)、`插`(insert)、`找`(find)、`含?`(contains)、`连接`(join)。

**高阶方法**（通过 `引 "../lib/数组.xt" 予 A` 引入标准库）：`A.映射`(map)、`A.过滤`(filter)、`A.归纳`(reduce)、`A.查找`(find)、`A.每个?`(every)、`A.存在?`(some)、`A.排序`(sort)、`A.去重`(dedup)、`A.反转`(reverse)。

> 完整教程、详细示例与输出请参阅：[06_数组编程指南.md](./06_数组编程指南.md)

### 6.4 字符串方法

*   包含获取与操作如 `拼接` (concat)、`截取` (substring)、`包含?` (contains)、`替换` (replace)、`分割` (split) 等操作，以及 `获取字符` 和 `长度` 等属性读取。

### 6.5 字典方法

*   支持 `设置` (set)、`获取` (get)、`包含?` (contains)、`键集` (keys) 等。

## 7. 玄铁类型系统哲学设计
具体看 [02_类型系统哲学设计.md](./02_类型系统哲学设计.md)
是由两套方案组成：

1. 多档开关——在执行编译指令时加入指令参数`--严格、--禁止、--默认`  即可切换编译器语法检查等级，具体分级制度如下
   - 错误等级：
     - 宽松级：未使用的变量　 | 　静默处理
     - 警告级：隐式类型、若抑否逻辑表达式出现多个 `否` 编译器会只采用第一个否、隐式动态、可推断未声明　 | 　编译期输出警告，强烈建议开发者修复这个问题
     - 错误级：完全错误的语法　 |　编译器完全退出并中断编译任务
   * 默认：按原分级处理（宽松→静默，警告→提示，错误→中断）
   * 严格：所有等级提升一档（宽松→警告，警告→错误，错误→错误）
   * 禁止：所有非错误提升为错误（宽松→错误，警告→错误，错误→错误）
2. 项目配置文件——在项目根目录放一个 `玄铁.配置.toml`
   ```
   [编译]
   严格度 = 禁止
   警告即错误 = ["隐式类型", "未使用变量"]
   忽略警告 = ["注释格式"]

   [版本]
   玄铁版本 = "0.18"
   ```

## 8. 网络与并发（v0.2.0 新增）

> 详细教程、完整示例与性能建议请参阅：[08_网络与并发编程指南.md](./08_网络与并发编程指南.md)

### 8.1 并发关键字

玄铁采用 M:N 线程池模型，类似 Go 的 goroutine。

| 关键字 | 说明 |
|--------|------|
| `异步 { ... }` | 将代码块提交到线程池异步执行，返回任务句柄 |
| `等待(任务)` | 阻塞等待异步任务完成并返回结果 |
| `并行 { {a}, {b} }` | 并发执行多个代码块，返回结果数组 |
| `道(容量)` | 创建带缓冲的并发通道 |

**已知局限**：嵌套在回调函数内的异步块暂不支持捕获外部变量（详见 08 指南）。

### 8.2 网络原语

| 关键字 | 说明 |
|--------|------|
| `求(url)` | HTTP GET 请求，返回 `结果<字符串>` |
| `连(地址)` | TCP 连接，返回 `结果<Socket>` |
| `听(端口, 回调)` | TCP 监听，新连接触发回调 |

### 8.3 Socket 方法

| 方法 | 说明 |
|------|------|
| `.读()` | 读取数据返回字符串 |
| `.写(内容)` / `.发(内容)` | 发送数据 |
| `.关()` | 关闭连接（通常无需手动调用，ARC 自动管理） |

### 8.4 快速示例

```text
// HTTP 请求
设 resp = 求("http://api.example.com/data")
若 resp.成功 { 示(resp.值) }

// TCP 服务端
听(8000, 函(流) {
    流.写("欢迎！")
    示(流.读())
})

// TCP 客户端
设 conn = 连("127.0.0.1:8000")
若 conn.成功 {
    conn.值.发("hello")
}
```

## 9. 玄铁渲染库（v1.1）

> 详细教程、完整 API 参考与示例代码请参阅：[09_渲染库编程指南.md](./09_渲染库编程指南.md)
>
> 底层基于 raylib 6.0，专注桌面 2D 渲染。引入方式：`引 "../lib/渲染/渲染.xt" 予 R`

### 9.1 核心 API 概览

| 类别 | 代表性 API |
|------|-----------|
| 窗口 | `R.初始化窗口(宽, 高, "标题")`、`R.窗口应关闭()`、`R.关闭窗口()` |
| 帧控制 | `R.开始绘图()`、`R.结束绘图()`、`R.清背景(r, g, b, a)` |
| 形状 | `R.绘直线`、`R.绘矩形`、`R.绘圆`、`R.绘椭圆`、`R.绘三角`、`R.绘多边形` |
| 文字 | `R.绘文字`、`R.绘帧率`、`R.量文字` |
| 字体 | `R.加载字体`、`R.加载字体精准`、`R.绘字体文字`、`R.释字体` |
| 纹理 | `R.加载纹理`、`R.绘纹理`、`R.绘纹理Ex`、`R.释纹理` |
| 输入 | `R.键按下?`、`R.键刚按下?`、`R.鼠标按下?`、`R.取鼠标X/Y` |
| 音频 | `R.加载音效`、`R.播音效`、`R.加载音乐流`、`R.播音乐流` |
| 工具 | `R.取随机数`、`R.取运行时间`、`R.等待` |

### 9.2 字体模式选择

| 模式 | API | 纹理 | 适用场景 |
|------|-----|------|---------|
| 全范围 | `加载字体(路径, 字号)` | ~8192×4096 | 任意中文，不预知字符集 |
| 精准 | `加载字体精准(路径, 字号, "参考文本")` | ~256×256 | 已知渲染内容，追求清晰与轻量 |

**注意**：仅支持 `.ttf`/`.otf`，不支持 `.ttc`（raylib 限制）。

### 9.3 快速示例

```text
引 "../lib/渲染/渲染.xt" 予 R

R.初始化窗口(800, 600, "示例")
设 字体 = R.加载字体精准("assets/font.ttf", 24, "你好世界")

当 非 R.窗口应关闭() {
    R.开始绘图()
    R.清背景(20, 20, 40, 255)
    R.绘字体文字(字体, "你好世界", 100, 100, 24, 2, 255, 255, 255, 255)
    R.绘帧率(700, 20)
    R.结束绘图()
}

R.释字体(字体)
R.关闭窗口()
```
