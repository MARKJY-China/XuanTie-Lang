# 玄铁编译器功能实现清单 (XuanTie Compiler Implementation Checklist)

本文档列出了玄铁编译器需要支持的所有核心语法、功能及其在**自举编译器 (Bootstrapped Compiler)** 中的当前实现状态。

> **状态定义**：
> - [x] **已实现**：语法已解析，IR 已生成，且在 `入口.xt` 的自举测试中稳定运行。
> - [ ] **待实现**：自举编译器尚不支持此功能，或仅在种子编译器 (Go 版) 中支持。

## 1. 基础语法与核心词法
- [x] 变量声明 (`设`)
- [x] 常量声明 (`常`) —— *注：自举版目前按变量处理，尚未强制不可变性*
- [x] 整数、布尔、空值字面量
- [x] 字符串字面量 (支持转义字符)
- [x] 字符串插值 (`"#{表达式}"`)
- [x] 算术运算 (`+`, `-`, `*`, `/`, `%`)
- [x] 比较运算 (`==`, `!=`, `<`, `>`, `<=`, `>=`)
- [x] 逻辑运算 (`且`, `或`, `非`)
- [x] 赋值语句 (`=`)
- [x] 成员访问 (`.`)
- [x] 下标访问 (`[]`)
- [x] 类型判断 (`是`)

## 2. 流程控制
- [x] 条件分支 (`若`, `否`)
- [ ] 多重分支 (`抑`)
- [x] 条件循环 (`当`)
- [ ] 无限循环 (`循`)
- [ ] 容器遍历 (`遍历 ... 于 ...`)
- [ ] 循环控制 (`断`, `续`)
- [ ] 模式匹配 (`匹配`)
- [ ] 程序终止 (`终`) —— *注：用于立即结束进程并返回状态码，如 `终 0`*

## 3. 函数与作用域

- [ ] 匿名函数/闭包

- [x] 块级作用域
- [x] 自动引用计数 (ARC) 内存管理

## 4. 面向对象 (OOP)

- [x] 实例方法与属性
- [x] `此` (This) 指针
- [x] 类继承 (`承`)
- [ ] 方法重写标记 (`覆`) —— *注：解析已支持，语义检查与分发优化待增强*
- [ ] 访问控制 (`公`, `私`, `护`) —— *注：解析已支持，编译器后端尚未实施访问限制*
- [x] 运算符重载 (魔术方法如 `_加_`, `_等_`)

## 5. 高级数据结构
- [x] 数组字面量 (`[...]`)
- [x] 字典字面量 (`{...}`)

- [ ] 模块引入 (`引`) —— *注：目前自举版暂由入口脚本预加载，编译器内部尚未实现递归编译*

- [x] 运行时基础库 (xt_runtime)
- [ ] 系统 I/O (文件、网络)
- [ ] 并发通道 (`道`)

## 8. 编译器架构演进与未来规划 (Future Evolution)
- **多趟编译与语义分析 (Multi-pass Compilation)**：引入独立的类型推导和作用域检查阶段，摆脱目前单趟遍历的局限，提升复杂代码的验证能力。
- **静态类型推导与去虚化 (Devirtualization)**：在编译期推断对象类型。如果确认为基础类型，则直接生成 LLVM 原生运算指令（如 `fadd`），绕过动态分发（魔术方法），大幅提升性能。
- **高级内存管理 (Advanced ARC & Escape Analysis)**：引入逃逸分析，将不会逃逸的对象分配在栈上而非堆上以减少内存碎片；引入弱引用（`弱`）解决 ARC 的循环引用问题。
- **错误报告体验 (Diagnostics & DX)**：实现带颜色高亮、具体错误位置（行列号）指出及修复建议的现代化编译错误提示（类似 Rust）。
- **模块化构建系统 (Build System)**：完善 `引` 机制，支持多文件、循环依赖解析，最终演化出玄铁专属的包管理与构建工具。
- **跨平台与 FFI 增强**：完善对 C 动态库的无缝调用（`外`），并利用 LLVM 完善对 Windows/Linux/macOS 及 WebAssembly 的交叉编译支持。
- **调试信息 (Debug Info)**：在生成的 LLVM IR 中嵌入 DWARF 信息，使得开发者能够使用 GDB/LLDB 甚至现代 IDE 源码级调试玄铁程序。
